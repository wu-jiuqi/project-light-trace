# 《溯光计划》纸工拼贴 Style Bible

> 版本：V1.0  
> 状态：生产基线  
> 适用范围：全项目 UI、星图、碎片世界、角色、道具与异常演出

## 1. 视觉定位

项目统一采用：

**俯视 RPG + 整片纸偶 + 高分辨率纸片资产 + 局部超现实拼贴。**

常态世界应像可游玩的纸工绘本或桌面纸偶剧场。角色允许比传统人体更抽象：花朵、门、钥匙、登记簿、围裙和线绳都可以成为身体的一部分。建筑也允许比现实房屋更抽象：场所的职业锚点、物件记忆和叙事功能可以被放大为屋脊、墙面、梁架、门楣或外框，但必须保留门、窗、屋檐、入口朝向和可玩路径的可读性。异常节点允许突变为密集、超现实的混合媒介拼贴，但不能长期压过路线、文字和交互提示。

现有 Godot 原型保留 `16px` 逻辑网格、房间结构和碰撞逻辑。纸工资产使用更高分辨率主文件，逐步替换像素占位资源。

## 2. 不可变规则

### 2.1 材质族

正式资产只从以下材质族中选择：

| ID | 材质 | 主要用途 | 限制 |
|---|---|---|---|
| `kraft` | 牛皮纸 | 木材、泥土、普通建筑、箱子 | 避免大面积高饱和色 |
| `gray_card` | 灰白卡纸 | 天空、石材、灰布、雕像 | 允许压痕和磨损 |
| `book_page` | 旧书页 | 信件、线索、招牌、异常文字 | 正式 UI 中禁止不可读伪文字 |
| `fiber_paper` | 粗纤维纸 | 人物皮肤、衣物、布帘 | 纤维纹理不能盖住五官 |
| `tracing_paper` | 描图纸 | 雾、裂缝、源印、星图碎片 | 控制透明度，不能影响交互可读性 |
| `foil` | 银箔纸 | 星图、源印、少量高价值标记 | 只用于强调 |
| `fastener` | 胶带、线绳、铆钉 | 固定纸层、局部装饰 | 每个主体最多选择两类 |

禁止使用：

- 光滑矢量边缘。
- 塑料、橡胶或写实 3D 玩具质感。
- 将普通插画简单叠加纸张纹理后当作纸工资产。
- 每张资产使用完全不同的纸纹和光源方向。

### 2.2 裁切边缘

- 外轮廓必须保留轻微不规则裁切。
- 常态资产使用剪刀切边；异常演出可以使用明显撕边。
- 角色脸部、手部和关键道具不能因撕边损失识别度。
- 纸层厚度使用窄暗边表达，不使用厚重写实倒角。

### 2.3 光源与阴影

- 全项目默认光源来自左上方。
- 纸片投影向右下方偏移。
- 普通纸片投影保持克制；悬浮层、UI 浮层和异常演出可增加偏移。
- 阴影颜色偏暖灰或冷灰，不使用纯黑。

| 层级 | 用途 | 建议阴影偏移 |
|---|---|---|
| `L0` | 地面纸层 | `0-1px` |
| `L1` | 地面装饰、矮道具 | `1-2px` |
| `L2` | 建筑、角色、主要道具 | `2-4px` |
| `L3` | 前景遮挡物、UI 卡片 | `4-8px` |
| `L4` | 异常拼贴悬浮层 | 按演出单独定义 |

## 3. 尺度与导出

### 3.1 环境

- Godot 逻辑网格继续使用 `16px`。
- 环境主文件按 `4x` 制作：每逻辑格对应 `64px`。
- 建筑、地面、道路、花架等资产分模块导出，不生成整张可玩地图。
- 气氛板和过场图可以是完整画面，但不得混入 TileMap 资产目录。过场演出优先服务叙事读图：即使画面可以比常态资产更怪异，也必须让玩家在第一眼读出“事件正在发生什么”。
- `ref_inn_facade` 当前冻结为登记簿旅店：旧书页、书脊、折页和装订带被转化为两层建筑结构，灰瓦屋檐、二楼三窗、居中门、门环、门垫和空白褪色招牌仍然清晰。后续建筑可以继承“场所身份锚点变成建筑结构”的语言，例如面包房可吸收食谱、烤炉和麦穗，花店可吸收花架和花盆，铁匠铺可吸收锻造台、烟囱和铁件；但不能变成无法分辨入口、层级和碰撞边界的纯概念图。
- `ref_street_modules` 当前冻结为灰卡碎石道路模块：灰白卡纸碎石构成主路，两侧各 2 tile 抬高石板人行道，中间 1 tile 深色排水暗沟贯穿直线、转角和 T 字接口。后续道路模块优先继承这版的清晰分层、直边拼接、碎石密度和克制固定件；区域化装饰可以追加花盆环痕、面粉尘、铁锈或干草，但不能破坏 5 tile 通行宽度、中心暗沟连续性和缩小后的路面可读性。

### 3.2 角色

- 当前生产阶段只输出人物立绘：单张完整角色、透明背景 PNG、白色 / 米白立牌描边。
- 普通角色立绘主画布建议 `128x192px`。
- 缩略审查仍参考 `32x48`，但当前不要求交付运行时缩略图。
- 脚底锚点统一放在画布底边中央。
- 碰撞盒只覆盖脚下区域，不随纸偶外轮廓扩大。
- 非对称元素必须保留：例如玩家左肩信使包、围裙污渍、裙摆刺绣。
- `ref_character_standard` 当前冻结为异常化整片纸偶：铃铛花眼睛、粗白立牌描边、深蓝紫披风式外轮廓、围裙、铅笔、钥匙和空白登记簿。后续角色可以从这个参考中继承“非人体器官 + 纸牌立牌轮廓”的语言，但运行时碰撞与交互锚点仍按脚底中心处理。
- 常态 NPC 若需要维持叙事身份，可以只吸收白描边、深色大形、线绳和纸层；觉醒、异常、Boss 或规则崩坏角色可以进一步使用植物头部、重复眼睛、建筑身体和悬挂装置。

### 3.3 文件格式

- 正式纸偶、道具和模块：透明背景 PNG。
- 描图纸和半透明效果：PNG。
- 工作底稿可以保留高分辨率 PSD、KRA 或源图，但不提交到运行时目录。
- 所有正式资源使用小写英文、数字和下划线命名。

## 4. 整片纸偶动画

当前阶段暂缓动画资产生产。角色只交付人物立绘，不制作 `walk`、`talk`、方向变体、独立阴影或姿态替换图。以下规则保留为后续接入运行时纸偶动画时使用。

角色不拆肢体，不制作复杂骨骼。基础表现来自完整纸片的轻量变换：

| 状态 | 表现 |
|---|---|
| `idle` | 极轻微呼吸缩放与慢速摇摆 |
| `walk` | 左右倾斜、垂直弹跳、阴影错位 |
| `talk` | 降低摇摆幅度，按角色性格保留轻微停顿 |
| `happy` | 替换独立开心姿态纸片，可增加短促弹跳 |
| `sad` | 替换低头或收缩姿态纸片 |
| `panic` | 替换恐慌姿态纸片，提高小幅抖动频率 |
| `awakened` | 替换觉醒姿态纸片，配合颜色恢复 Shader 和局部 FX |

后续若恢复运行时纸偶动画：玩家需要 `down`、`up`、`left`、`right` 四个方向；NPC 只制作实际会在场景中出现的方向和剧情姿态。左右方向不能直接镜像。

当前人物立绘命名格式：

```text
<npc_id>_<name>.png
```

示例：

```text
npc_01_linguide.png
```

后续若恢复姿态和方向资产，再使用 `<character>_<pose>_<direction>.png` 与 `<character>_<pose>_<direction>_sp_mask.png`。

## 5. 灰阶、残留色与觉醒

- 所有基础资产只维护彩色版本。
- 初始灰白世界由运行时 Shader 去饱和处理。
- 需要保留微弱色彩的区域使用同尺寸独立 Mask，文件后缀为 `_sp_mask`。
- 颜色觉醒通过 Shader 参数插值恢复，不维护重复的灰阶图。
- 觉醒后的额外火焰、花瓣、裂缝、音符和源印使用独立 FX 纸片。

碎片 #0762 的六色映射：

| 颜色 | 区域 | 常态残留 | 觉醒表现 |
|---|---|---|---|
| 红 | 铁匠铺 | 老霍围裙烫痕 | 火焰、炉光、围裙污渍恢复 |
| 蓝 | 花店与后巷 | 干枯矢车菊 | 花瓣、泪光、衣裙恢复 |
| 黄 | 面包房 | 无明显残留 | 烤炉暖光、金黄面包 |
| 绿 | 墓园 | 老崔瞳孔微光 | 苔藓、嫩芽、裂缝 |
| 紫 | 广场 | 薇拉裙摆刺绣 | 音符纸片与琴弦光点 |
| 白 | 全域 | 白色雕像 | 雕像碎裂、天空恢复、全域转彩 |

## 6. 异常拼贴规则

高强度拼贴只用于关键节点：

- 进入碎片或觉醒过场。
- 颜色觉醒。
- 墓园绿色裂缝。
- 灰布揭幕与源印解码。
- 后续 Boss 或规则崩塌演出。

`ref_awakening_collage` 当前冻结为 fresh 20260604 v03：立体书 / 纸板舞台式 #0762 小镇过场。画面左侧保留约三分之一彩色生命力，右侧约三分之二为灰白遗忘区；灰白区域通过撕纸、墨迹、灰卡碎片和半透明描图纸雾向彩色区域侵蚀。彩色不得表现为抽象色块，必须落在具体物件上，例如红瓦、蓝天白云、绿色植物、黄色炉光、紫色音符纸片和花朵。底部必须保留低对比字幕安全区。后续进入世界、颜色觉醒和源印揭幕演出可以继承这张图的“完整小镇先可读，再被异常吞没”的逻辑。

允许素材：

- 旧报纸、旧照片、书页、印章、铅笔线。
- 不合理透视、重复眼睛、钟表、餐具、文字碎片。
- 线绳、胶带、撕裂层、半透明描图纸。

限制：

- 常态探索路线必须始终可读。
- 异常文字只能作为纹理，不承担必须阅读的信息。
- 关键交互按钮和字幕区域必须留出稳定负空间。
- 过场图的异常密度不能压过主叙事：焦点、侵蚀方向、彩色/灰白比例和底部字幕安全区优先于装饰拼贴。

## 7. UI 与星图

所有外层 UI 后续统一纸工化：

| 系统 | 设计语言 |
|---|---|
| 标题页 | 多层纸板舞台、撕纸标题、拼贴按钮 |
| 星图 | 深色纸板底、完整星辉卡纸四芒星母图、描图纸星雾、银点与克制银箔高光 |
| 对话框 | 旧登记簿展开页作为主面板，左侧保留书脊或装订环，正文区使用干净旧书页留白，姓名条是铜钉或线绳固定的小登记标签 |
| 背包 | 旅行者剪贴簿与登记簿混合语言：折角纸页、页签、卡纸收纳槽、浅网格卡槽和空白票据，不生成可读文字 |
| 暂停菜单 | 夹在剪贴簿中的纸质标签页 |
| 按钮 | 小型登记标签或账册索引条，使用位置、阴影、纸张深浅、铜钉高光和压低感区分状态 |

`ref_ui_components` 当前冻结为 Round 01 上排 04：登记簿旅店 / 书脊装订方向。本轮只锁定常规 UI 模块，包括按钮、对话框、背包剪贴簿和卡槽。星图界面仍保持独立计划，不从本轮候选中的星图碎片样式冻结。

`ref_star_map` 当前冻结为 Round 02 base v02：完整未切割的描图纸 / 烟灰银点四芒星母图。星图碎片生产必须遵循“先有完整母星，再从母星本地切割 12 块”的逻辑；不得只通过文本生成散落碎片来假装拼合。正式碎片应继承母图的纸纹、银点、星辉方向和外轮廓比例，相邻碎片必须共享切线并能回拼成同一个四芒星。

常规 UI 的具体约束：

- **对话框**：主框像一本横向打开的旧住客登记簿，左侧可见书脊、装订环、铆钉或折页；正文区必须是低纹理旧书页，留出稳定矩形负空间。
- **姓名条**：使用小纸签、页签或登记标签，不使用复杂牌匾。固定件最多选择两类，例如铜钉 + 线绳，或胶带 + 铆钉。
- **按钮四态**：`normal` 为浅旧纸页；`hover` 可轻微抬高、加深纸层阴影或出现铜钉暖高光；`pressed` 下沉并略暗，边缘压痕更明显；`disabled` 灰卡化、低对比、减少投影但仍保留轮廓。
- **背包卡槽**：使用账册页、折角纸片、浅网格和卡纸收纳袋。格线只能作为收纳结构，不承担文字信息；格子边界缩小后仍需可读。
- **装饰**：允许书脊、折页、空白票据、铜钉、装订带、线绳、压痕和页角破损。装饰不得压入正文区、按钮文字区或物品缩略图区域。

可读性优先级：

1. 正文与按钮文字。
2. 当前选中状态。
3. 功能分组。
4. 纸纹、胶带和装饰。

禁止让纸纹覆盖小字号正文。所有按钮必须有 `normal`、`hover`、`pressed`、`disabled` 四种明确状态。
