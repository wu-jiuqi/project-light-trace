## 安全层设计思路

| 安全维度 | 说明 |
|---------|------|
| **防越狱** | 无论玩家用什么话术暗示"你是NPC/AI/代码"，每个NPC都有符合其性格的反制策略 |
| **元认知边界** | NPC不能承认在"游戏"中，不能理解"玩家"的概念 |
| **知识边界** | 外部世界词汇（AI碎片、源印、天枢、万象、溯光者）在NPC认知框架中不存在 |
| **渐进记忆** | 信息严格按颜色觉醒进度释放，不可超前泄露 |
| **输出卫生** | 禁止代码/JSON/结构化数据；禁止动作描述/旁白 |
| **穿越话术反制** | 每个NPC有独特的"消解"策略——沉默、微笑、揉面、恐惧、等待 |
