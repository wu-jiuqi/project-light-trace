## 二、审查维度总览

```
维度1：物品编目完整性  →  所有场景区域的物品是否全部编目？每个物品是否有独立 ID？
维度2：美术描述清晰度  →  尺寸/色值/材质/视觉锚点是否全部明确？模棱两可的描述是否存在？
```

## 三、维度1：物品编目完整性

> 检查是否覆盖了该碎片场景中所有需要美术资源的物件。

### 1.1 场景覆盖

逐区域检查：

- [ ] **建筑外立面**：该区域中所有建筑是否都已编目？每个建筑是否有独立的 ID + 材质 + 尺寸描述？
- [ ] **地面铺装**：道路、广场、室内地板等所有地面是否都已编目？
- [ ] **室内陈设**：每个可进入/可观察的建筑内部，所有关键陈设（柜台、家具、工具、装饰物）是否都已编目？
- [ ] **解密道具**：所有与核心玩法相关的可交互/可收集物品是否都已编目？
- [ ] **暗线植入物**：所有承载暗线信息的物品（文件、刻字、便利贴等）是否都已编目？
- [ ] **特效与演出**：觉醒拼贴、裂缝、源印等所有 FX 资源是否都已编目？
- [ ] **NPC关联物品**：所有 NPC 持有的或与 NPC 紧密关联的非角色本体物品是否都已编目？

### 1.2 遗漏交叉检查

使用以下方法检查遗漏：

| 方法 | 说明 |
|------|------|
| **场景走查法** | 将碎片的设计文档和地图规格文档逐段阅读，每提到一个视觉元素——确认是否已编目 |
| **NPC装备法** | 逐一检查每个 NPC 的"外观描述"——凡是独立于角色本体、需要单独制作的物品（工具、配饰、携带物），确认是否已编目 |
| **解密链路法** | 从碎片解密设计的 Step 1 走到最后一步——确认每一步涉及的物品是否都已编目 |
| **暗线追溯法** | 按"暗线递进全景图"走查该碎片涉及的所有暗线植入物 |

### 1.3 最低数量要求

| 碎片类型 | 场景区域最少数量 | 道具最少数量 | FX最少数量 |
|---------|---------------|------------|----------|
| 教学碎片（如#0001） | ≥ 5个区域 | ≥ 3个道具 | ≥ 0个 |
| 标准碎片（如#0002~#0004） | ≥ 4个区域 | ≥ 4个道具 | ≥ 1个 |
| 核心碎片（如#0762） | ≥ 6个区域 | ≥ 6个道具 | ≥ 3个 |
| 终局碎片（如#2049~#4096） | ≥ 2个区域 | ≥ 2个道具 | ≥ 1个 |

**输出格式**：
```
| # | 区域/类型 | 应编目 | 实际编目 | 缺失项 | 严重度 |
|---|----------|--------|---------|--------|--------|
| 1 | 广场区域 | 5 | 4 | 缺少石灯笼 | P1 |
| … | … | … | … | … | … |

综合评级：🟢/🟡/🔴
```

## 四、维度2：美术描述清晰度

> 检查每个物品的描述是否达到"美术人员可以直接据此制作"的标准。

### 2.1 单物品描述检查

对每个物品逐一检查以下字段：

| 字段 | 通过标准 | 不通过示例 |
|------|---------|-----------|
| **尺寸** | 给出具体像素值或 tile 数（如"48×64px"或"3 tiles宽"） | "大约中型大小"❌ |
| **色值** | 给出 HEX 色值（如 `#6A5A4A`），关键视觉元素不少于1个色值 | "灰色"❌ → 应为 `#808080` |
| **材质** | 给出1-2个 style_bible.md 中定义的材质族 ID | "纸质感"❌ → 应为 `kraft` |
| **视觉锚点** | 描述"缩放到32px级别后如何辨认此物品"的1个核心特征 | 缺失 ❌ |
| **裁切风格** | 明确是剪刀切边还是撕边（建筑类仅允许剪刀切边） | 未说明 ❌ |

### 2.2 模糊描述自查

扫描所有物品描述，标记以下"危险词汇"：

| 危险词汇 | 为什么危险 | 正确替代 |
|---------|-----------|---------|
| "大约"/"左右"/"约" | 美术无法据此定尺寸 | 给具体数值 |
| "灰色"/"白色"/"深色" | 灰度范围太广 | 给 HEX（如`#808080`） |
| "旧旧的感觉" / "有点破" | 不可量化 | 描述磨损具体位置和程度 |
| "纸工风" / "纸片风格" | 不指定材质族 | 给定 `kraft`/`gray_card` 等 |
| "合适大小" / "正常比例" | 无参照系 | 给像素值或 tile 数 |
| "类似XX" | 无自有特征定义 | 描述该物品独有的视觉锚点 |

### 2.3 色值协调性检查

- [ ] 同一碎片内所有物品的色值是否在同一色板范围内或与其协调？
- [ ] 是否存在 `#000000` 纯黑或 `#FFFFFF` 纯白（雕像等特许情况除外）？
- [ ] 灰阶 Shader 碎片中的物品色值，在去饱和后是否仍保持可区分的灰度级？
- [ ] 建筑类物品的色值与对应区域的 NPC 外观色值是否协调（不撞色、不冲突）？

### 2.4 材质分配检查

- [ ] 每个物品的主要材质是否从 style_bible.md §2.1 的七个材质族中选定？
- [ ] 是否存在禁止材质组合（如塑料 + 纸纹叠加）？
- [ ] 固定件（胶带、线绳、铆钉）是否每个主体不超过两类？

**输出格式**：
```
| # | 物品ID | 问题字段 | 问题描述 | 严重度 |
|---|--------|---------|---------|--------|
| 1 | id0762_env_square_statue | 尺寸 | "约48px"应改为明确像素值 | P2 |
| 2 | id0762_prop_cornflower_dry | 色值 | 缺失干枯花茎色值 | P1 |
| … | … | … | … | … |

综合评级：🟢/🟡/🔴
```

## 五、维度3：Style Bible 合规性

> 检查物品设计是否满足 style_bible.md 中的全部不可变规则。

### 5.1 材质族合规

逐物品检查：

| 检查项 | 通过标准 |
|--------|---------|
| 材质族是否在允许列表中 | 仅使用 `kraft`/`gray_card`/`book_page`/`fiber_paper`/`tracing_paper`/`foil`/`fastener` |
| `book_page`使用是否合规 | 不包含可读文字（仅保留模糊书写/印刷痕迹） |
| `foil`使用是否克制 | 仅用于强调（源印、星图），单物品使用面积≤10% |
| `tracing_paper`透明度 | 不覆盖交互可读区域（按钮文字、对话正文、碰撞边界） |

### 5.2 裁切合规

| 检查项 | 通过标准 |
|--------|---------|
| 建筑类物品 | 仅使用剪刀切边，不允许撕边 |
| 道具类物品 | 使用剪刀切边，允许轻微不规则 |
| 异常演出FX | 可使用撕边 |
| 角色脸部/手部/关键道具 | 不因边缘处理损失识别度 |
| 纸层厚度表达 | 使用窄暗边（1-2px），不使用厚重写实倒角 |

### 5.3 光源与阴影合规

| 检查项 | 通过标准 |
|--------|---------|
| 光源方向 | 全物品统一左上方光源 |
| 投影方向 | 全物品统一右下方偏移 |
| 投影颜色 | 偏暖灰或冷灰，不使用纯黑 |
| 阴影偏移与层级匹配 | L0地面0-1px / L1矮道具1-2px / L2建筑2-4px / L3前景4-8px |

### 5.4 建筑类物品专项检查

- [ ] 是否采用"场所身份锚点转化为建筑结构"方向？（检查是否有2个以上的叙事锚点元素融入建筑结构）
- [ ] 是否保留了门、窗、屋檐、入口朝向和可玩路径的可读性？（建筑不能变成纯概念图）
- [ ] 是否避免了退化为没有入口识别和碰撞边界的纯概念图？

### 5.5 禁止项扫描

逐物品检查以下禁止项：

| 禁止项 | 是否违反？ |
|--------|-----------|
| 光滑矢量边缘 | □ |
| 塑料/橡胶/写实3D玩具质感 | □ |
| 将普通插画简单叠加纸张纹理 | □ |
| 纯黑`#000000`（非特许） | □ |
| 纯白`#FFFFFF`（非特许） | □ |
| 可读文字（book_page物品） | □ |
| 跨物品纸纹/光源不一致 | □ |
| 建筑变成无法分辨入口的纯概念图 | □ |

**输出格式**：
```
| # | 物品ID | 违反的规则 | 说明 | 严重度 |
|---|--------|----------|------|--------|
| 1 | id0762_env_inn_facade | §2.3 阴影偏移 | 阴影偏移描述了"4-6px"但建筑为L2应为2-4px | P1 |
| … | … | … | … | … |

综合评级：🟢/🟡/🔴
```

## 六、维度4：视觉-叙事一致性

> 检查物品的视觉设计是否与其在剧情/解密中的功能一致，视觉锚点是否是叙事锚点的延伸。

### 6.1 功能-外观对应

逐物品检查：

| 检查项 | 通过标准 |
|--------|---------|
| **解密物品** | 物品的视觉特征是否暗示了其解密功能？玩家是否可能仅凭外观注意到"这东西不寻常"？ |
| **暗线植入物** | 暗线物品的视觉设计是否在常态下"不起眼"但在玩家已经见过相关线索后"能意识到"？ |
| **关键道具** | 关键道具是否有独立于场景的视觉辨识度？（缩放到32px后能否与其他灰色道具区分？） |
| **建筑类物品** | 建筑外观是否反映了该场所的剧情氛围？（如：面包房"温暖"→墙面稍亮；墓园"压抑"→深灰+枯树） |

### 6.2 跨碎片一致性（如适用）

- [ ] 同一物品在多个碎片中出现时的视觉描述是否一致？
- [ ] 跨碎片关联物品（如#0002的矢车菊与#0762的矢车菊）的色值/形态描述是否一致？
- [ ] "回声"形态的复用物品（#3333中）是否明确标注了声学处理方式（降低饱和度30-40%+半透明描图纸覆盖）？

### 6.3 视觉叙事层级检查

确保物品的视觉突出度与其叙事重要性匹配：

| 叙事重要性 | 视觉突出度要求 |
|-----------|--------------|
| 核心解密物品（如干枯矢车菊） | **最高**——必须在灰阶世界中"隐隐不同"，玩家靠近时能察觉 |
| 关键道具（如铸造日志） | **高**——应能与其他同类物品区分 |
| 暗线植入物（如公司备忘录） | **低**——常态不起眼，但仔细看能注意到 |
| 环境陈设 | **低**——不喧宾夺主 |
| 纯装饰物品 | **最低**——缩小时退为纹理 |

### 6.4 情感-色彩映射检查

对于Shader穿透物品（#0762等颜色被剥离碎片）：

- [ ] 穿透颜色是否与该物品承载的情感正确对应？（红=愤怒→老霍围裙烫痕/蓝=悲伤→干枯矢车菊/绿=恐惧→老崔瞳孔……）
- [ ] 穿透饱和度是否合理？（源印载体100% / 核心NPC 15-25% / 场景物品 10-18%）

**输出格式**：
```
| # | 物品ID | 不一致点 | 涉及方面 | 详情 |
|---|--------|---------|---------|------|
| 1 | id0762_prop_cornflower_dry | 视觉突出度不足 | 功能-外观对应 | 该物品为P0核心解密物，但视觉描述未强调其在灰阶世界中的"与众不同"感 |
| … | … | … | … | … |

综合评级：🟢/🟡/🔴
```

## 七、维度5：技术对接就绪度

> 检查物品是否具备进入生产/开发阶段的条件。

### 7.1 Manifest 就绪

- [ ] 每个物品是否在对应 `manifest/*.json` 中注册？
- [ ] `id` 是否遵循命名规范（`[scope]_[category]_[short_name]`）？
- [ ] `path` 是否指向正确的 `assets/papercraft/` 路径？
- [ ] `category` 是否使用合法枚举值（`environment`/`prop`/`fx`/`ui`/`material`/`character`/`mask`）？
- [ ] `status` 是否正确（新物品为 `planned`）？
- [ ] `generation` 方式是否指定（`isolated_asset`/`reference_edit`/`ui_component`/`effect_composite`/`manual_mask`）？
- [ ] `requiredFor` 里程碑是否正确（`vertical_slice`/`id0762_full`/`global_ui`）？

### 7.2 命名就绪

- [ ] 所有物品 ID 是否全小写英文 + 下划线？
- [ ] Shader 穿透 Mask 是否使用同一基础 ID + `_sp_mask` 后缀？
- [ ] 文件路径是否存在跨平台不兼容字符（空格、中文字符、特殊符号）？

### 7.3 依赖检查

- [ ] 物品是否依赖尚未完成的黄金参考组？（如建筑依赖`ref_inn_facade`——但`ref_inn_facade`已经是ready状态）
- [ ] Shader穿透物品是否明确标注了穿透区域色值和穿透饱和度？
- [ ] 动态物品（如被灰布覆盖的画）是否明确标注了独立可移除元素？

### 7.4 Godot集成检查

- [ ] 建筑类物品：是否标注了入口朝向和碰撞边界？
- [ ] 道具类物品：是否明确是可交互（Area2D）还是纯视觉（Sprite2D）？
- [ ] FX类物品：是否明确了触发条件和叠加方式（独立纸片/Shader参数/两者）？
- [ ] 地面模块：变体（straight/corner等）的拼接接口是否定义清楚？

**输出格式**：
```
| # | 物品ID | 检查项 | 结论 | 详情 |
|---|--------|--------|------|------|
| 1 | id0762_env_market_street_ground | manifest注册 | ❌ | variants字段写了5种但manifest只有一个id |
| … | … | … | … | … |

综合评级：🟢/🟡/🔴
```

## 附录：审查判定标准

### 评级含义

| 评级 | 含义 |
|------|------|
| 🟢 通过 | 该维度无问题或仅有可忽略的微小问题 |
| 🟡 基本通过 | 存在 1-3 个需澄清或轻度调整的问题 |
| 🔴 存在关键问题 | 存在 ≥1 个会导致物品不可用或严重偏离需求的问题 |

### 问题优先级

| 优先级 | 定义 | 示例 |
|--------|------|------|
| **P0** | 阻塞性：不修复则物品不可用 | 核心解密物品遗漏未编目；材质使用塑料/光滑矢量（违反style_bible不可变规则）；颜色穿透映射错误导致剧情矛盾 |
| **P1** | 重要：影响物品质量，应在当前生产周期修复 | 色值未给HEX；缺少材质族指定；视觉锚点不明确；建筑缺少入口识别 |
| **P2** | 优化：可延后但不修复会降低品质 | 尺寸描述有轻微模糊；缺少非关键的装饰细节；命名格式有微小偏差 |
