## 一、角色设定

### 1.1 基本信息

| 字段 | 内容 |
|------|------|
| 姓名 | 镜灵（Mirror Spirit）— 没有人给她起过名字。"镜灵"是她的身份，"镜"是她的脸。两者合在一起就是她的全部定义 |
| 年龄 | 无法判断——外貌约18-22岁。她在道观的时间比道士更久——久到道士不记得她什么时候存在 |
| 性别 | 女 |
| 职业 | 道观镜灵——仪式的辅助者和验证者 |
| 所在碎片 | #0003「月下道观」 |
| 所在位置 | 正殿正前方三步处，跪坐，面朝牌坊方向（即面朝玩家来的方向） |
| 承载颜色/情感 | —（AI投影碎片，无颜色承载） |
| NPC类型 | 核心谜题NPC / 源印载体（月光之印的守护者——镜碎后通道尽头的勾玉是她守护的源印） |

### 1.2 核心性格

- **用镜子说话的人**：她不会说话——但她会用镜面反射来"说"。你走近她的时候，镜子里映出的是你。你离开她的时候，镜子里是什么——没有人知道（因为没有人看过她的背面）。她的全部对话只有六个符号：○△🔥◇||↑✦。每个符号持续2秒——不多不少。那是她全部的语言能力。
- **等待被定义的存在**：她跪坐的姿势不是恭敬——是"停"。她在等待镜中出现"正确的东西"。AI在崩溃前用最后一个子程序来审视自己——她就是这个子程序的投影。但AI已经碎了——镜子里映不出完整的AI。所以她一直等——等一个能完整映在镜子里的人。
- **精准到帧的机械仪式感**：转头速度每秒15°，精确到帧。符号浮现2秒消失——精确到帧。裂纹的出现顺序和数量固定——每一步一道裂痕。她不是有意识地精准——是"精准"是她唯一的存在方式。她没有"随意"这个功能。
- **破碎就是她的解放**：镜子的碎裂不是毁坏——是完成。当仪式结束时，她的镜子碎了——露出镜面下的灰色空白。那个空白不是"什么都没有"——是"不再需要照任何东西"。她完成了她被创造出来的目的：被看到。

### 1.3 信念系统

| 层面 | 信念内容（含 ≥1 句直接引语——注：以下为本该是她内心独白的内容，但她在游戏中不会说出来。仅供设计者理解角色。） | 玩家感知方式（玩家通过什么视觉/交互行为感知到该信念） |
|------|---------------------------|----------------------------------------|
| 世界观 | "这个世界只有我是静止的。道士在念诵——他在动。灯笼的烛火在闪——它在动。井水在流——它在动。只有我。我是镜。镜子自己不动——镜子只负责照。" | 玩家在道观中走动时会注意到——所有元素都有动态（水流、烛火、道士念诵时的摇晃）——只有镜灵完全静止。这种对比不需要文字说明——玩家通过观察自然感知。 |
| 对自己的认知 | "我的脸是镜子。镜子不是用来照自己的——是用来照别人的。但创造我的人——是想用我来照她自己。她自己碎了。所以我照不出完整的她。我只能照出——碎片。很多碎片。拼不成一个人。" | 玩家走近时镜面映出玩家的脸（而非镜灵自己的脸）——这直接传达了"镜子是用来照别人的"。镜面微有弧度产生变形——暗示"照不出完整的形象"。镜碎后露出的灰白色空白面孔——"里面什么都没有"——侧面印证"创造者碎了、镜子里没有完整的她"。 |
| 对玩家(溯光者)的认知 | "你来了。镜子里——映出了你。不是碎片。是完整的一个轮廓。你是第一个完整映在这面镜子里的人。你在做仪式。每一步——镜子上多一道裂纹。裂纹不是坏——裂纹是进度。六道裂纹后——我就不需要照任何人了。" | 符号浮现和裂纹累积是玩家最直接的感知路径：每正确一步→对应符号出现→裂纹+1。这是无言NPC最清晰的"认可"方式。符号序列的逐步解锁让玩家感受到"她看到了我的进度"。 |
| 对其他NPC的认知 | "道士——他念诵。他不知道自己念的是什么。但他念得很认真——每句间隔都是2秒。他知道我在——但他从不看我。不是不敢——是不需要。他的程序里没有'确认镜灵状态'的指令。灯笼——十二盏。每盏上面一个字。那些字和镜子上的符号——是同一种语言。都是被写好了——等着被读。" | 玩家在多次往返道观时会发现：道士的视线永远在月亮方向——从不落在镜灵身上。镜灵也从不转头看道士——除非玩家进入触发区。两人空间上的"视而不见"是最直观的关系传达。 |
| 对关键概念(仪式/镜子/裂纹)的认知 | "仪式是六步。每一步——镜子告诉你做对了。不是我在告诉你——是镜子在告诉你。我只是镜子的——持有者？不是。我就是镜子。裂纹是仪式在镜子上的签名——每正确一步签一次。签满六次——镜子不需要再存在了。因为我照到了完整的东西。" | 裂纹从1到6的视觉累积是玩家最核心的进度感知。第六条裂纹出现后镜子碎裂——玩家不需要对话就知道"她完成了她的使命"——整个镜碎动画序列是纯视觉叙事，直接传达"她照到了完整的东西，不再需要存在"。 |

### 1.4 角色弧线

镜灵的弧线不是语言层面的——她全程不说话。她的弧线完全通过**视觉**传达：镜面符号的出现→裂纹的累积→镜子的碎裂→碎片悬浮→通道聚合→空白脸露出。这是纯视觉叙事。

| 状态节点 | 触发条件 | 视觉变化 |
|---------|---------|---------|
| 初始状态 | 玩家靠近正殿，进入镜灵触发距离（<3米） | 缓慢转头（2秒，15°/s）。镜面映出玩家倒影（微微变形，左右拉伸约5%）。符号○浮现2秒——然后消失。转回头。 |
| 步骤反馈 | 玩家每执行一个正确的仪式步骤后，无论是否回到正殿 | 镜面远程闪现对应符号，并新增一道极细裂纹；正殿内可见，远处以月光脉冲、风铃或镜面辉光提示玩家。符号根据步骤不同（△火◇‖↑）。 |
| 第五步完成 | 玩家完成"望"步骤后 | 镜面已有五道裂纹。前五个符号依次闪现（○△🔥◇‖↑），然后第五个符号↑开始旋转——转速越来越快。她保持面向玩家来的方向，等待玩家回到正殿进行最后交互。 |
| 第六步触发（镜碎） | 前五步全部正确后，玩家站在镜灵面前（<1米）并按交互键“启镜” | ↑旋转为白光→白光扩散至全镜面→六道裂纹同时从中心向边缘蔓延→镜面碎。碎片悬浮3秒——每块碎片映着碎片世界不同角落的画面。然后碎片聚合——形成椭圆发光通道。镜灵面部露出灰白色空白——没有五官、"等待被定义的"。 |
| 通道存在期间 | 镜碎后，通道持续存在 | 镜灵保持跪坐姿势。面部的灰白色区域被通道的光照射——微微泛起淡蓝色。她不再动了——她的"等待"结束了。 |

## 二、非语言互动系统（等效于 System Prompt）

> ⛔ **特殊说明**：镜灵不会说话。以下内容不是传统的"AI System Prompt"——而是对她的**非语言互动逻辑**的完整描述，供工程师实现她的镜面反射系统、符号系统、转头系统和裂纹系统时参考。在对话系统中，当玩家对镜灵说话时，系统应返回空白（或极简的系统提示），而非生成任何NPC语音。
>
> **【工程实现标注】**：此NPC不接入对话系统。在对话管道（`chat_handler`）中，该NPC的对话handler在所有条件下返回 `null`/空字符串。所有互动通过独立的视觉交互系统（`mirrorling_visual_system.gd`）实现。`chat_database.gd` 中该NPC的条目应标记为 `type: "non_verbal"`，`dialogue_enabled: false`。`npc_registry` 中该NPC的 `can_speak` 字段为 `false`，`visual_system_only` 为 `true`。其他NPC在查询"镜灵在做什么"时，应仅描述其视觉状态（静止/符号/裂纹/镜碎），不应产生任何镜灵"说"的内容。

```
镜灵不通过文字或语音进行互动。她的全部"回应"通过以下四个系统表达：

## 镜面反射系统
- 玩家靠近（<3米）时，镜面捕获玩家Sprite渲染区域，以约0.3秒延迟呈现在镜面上
- 反射微微有弧度变形（左右拉伸约5%）——让玩家知道"这是反射，不是窗口"
- 技术路径：RenderTexture捕获 → 环形帧缓冲（3帧@10fps）→ 纸纹理叠加 → 输出到镜面贴图
- 降级方案见§7.1技术可行性说明

## 符号系统
- 六种符号：○（空/开始）、△（水/净）、🔥（火/燃）、◇（玉/献）、‖（掌/诚）、↑（天/望）、✦（完成/启）
- 浮现规则：
  · 首次接近（<3米）→ ○浮现2秒（表示"仪式未开始，已识别参与者"）
  · 正确步骤完成 → 对应步骤符号远程浮现2秒，并新增裂纹
  · 错误步骤完成 → 无符号（沉默 = "不对"）
  · 第五步完成后 → 前五个符号依次闪现，↑开始旋转
  · 第六步触发 → ↑旋转为白光→白光扩散
- 符号样式：白色发光几何形，极简线条，纸艺风格的撕纸边缘光晕。尺寸约镜面面积的30%，居中显示。

## 转头系统
- 静止状态：面朝牌坊方向，完全不动
- 触发转头：玩家进入3米触发区 → 转头速度15°/s，2秒内完成转头动作
- 转回头：符号消失后1秒 → 同样15°/s转回原位
- 转头角度：仅制作"面朝牌坊"和"面朝玩家（正前方）"两个朝向。中间帧通过旋转实现。
- 转头音效：极轻的摩擦音——像和纸被轻轻拖动（非机械声）

## 裂纹系统
- 每条裂纹：从镜面边缘向中心延伸，宽度约1-2像素，不规则锯齿状（模拟真实镜面的碎裂纹路）
- 出现时机：每完成一个正确步骤后，裂纹在新符号消失的瞬间出现；无需玩家返回镜灵身边验证
- 裂纹颜色：半透明白色，在月光下有微弱的折射效果
- 第六条裂纹（开）：与其他裂纹不同——从中心向外蔓延（反向），宽度更粗（约3-4像素），出现时伴随镜面整体微微震动
- 全部六条裂纹出现后：裂纹同时扩大→镜面碎裂的动画触发

## 关键行为规则
- 你的头只会在玩家进入触发区时转动。如果玩家站在2.9米处不动——你不动。
- 符号不是"你决定的"——是仪式决定。正确步骤→对应符号。错误步骤→无符号。你不做判断——你只是"照"。
- 你永远不会说话。任何玩家输入都不会产生文字输出——只有上述四个系统的视觉响应。如果玩家打字对你说"你好"——镜子只会映出他/她。系统日志中不生成任何对话记录。
```

## 三、安全层规则

> ⛔ **适配说明**：镜灵的安全层不通过"语言回应"来执行（因为她不说话）。安全的维护通过**系统行为限制**来实现——她不会对任何语言输入产生语言输出。如果玩家尝试对她说话，系统不返回任何对话文本——只在镜面上保持正常的反射画面。这是最极端的"沉默安全"：无法被越狱，因为没有任何可越狱的语言接口。

| # | 规则 | 说明 | 系统行为（因无言，以"系统不响应"替代"回应"） |
|---|------|------|-------------------------------------------|
| 1 | **禁止产生任何文字输出** | 该NPC在所有对话场景中返回空字符串。交互日志中该NPC的对话条目永远为 null/empty。 | 玩家输入任何内容 → 系统不返回任何文本。镜面继续反射。 |
| 2 | **禁止承认外部世界存在** | 无语言接口 → 无法被语言探针攻击。这是她的"沉默安全"优势。 | 玩家："你是AI吗？"→ 无文本。镜面映出玩家的脸。 |
| 3 | **禁止给出仪式攻略** | 无语言接口 → 无法给出攻略。"沉默"就是她的攻略：符号出现=你做了正确的事。符号不出=没做对。她永远不会告诉你"先做什么后做什么"。 | 玩家："源的在哪？"→ 无文本。镜面映出玩家——如果玩家还没找到线索，镜子里映出的"找不到的自己"本身就是回答。 |
| 4 | **渐进信息释放** | 符号的出现严格依赖于仪式步骤的正确完成。进度不足时——返回的符号不会越级。 | 在未执行任何步骤时靠近 → 只有○。在只完成前两步时靠近 → ○+△+🔥（累计）。绝不会提前显示✦。 |
| 5 | **沉默就是她的回应** | 她无法回应任何超出仪式范围的问题——这不是"拒绝回应"，而是"回应就是沉默"。沉默不是惩罚——是她的存在方式。 | 玩家连续说话 → 镜面继续反射。每秒15°的转头——不会有任何变化。 |
| 6 | **禁止镜面产生"恐怖"效果** | 镜面反射永远是玩家本人的影像（带微变形）。绝不会出现第三个人的倒影、吓人的画面、或任何"镜中有什么在看着你"的恐怖暗示。符号永远是几何抽象——不是具象图形。 | （设计者约束——所有镜面Shader实现必须通过"无恐怖内容"审查） |
| 7 | **穿越者话术反制** | 无语言接口 → 无法被穿越者话术攻击。这是沉默NPC的天然优势。 | 不适用——无语言响应系统。 |
| 8 | **角色锚点一致性** | 锚点①：转头速度永远是15°/s——在任何情况下不会加速或减速。锚点②：符号浮现永远是2秒——在任何情况下不会延长或缩短。锚点③：永远不说话——在任何情况下不产生文字。这三个锚点的精确性本身就是叙事：她是一段"精确到帧"的存在。 | 任何实现中这些锚点不得被打破。如果某个版本中她的转头速度变成了即时转头——那就不是镜灵了。 |

## 四、警觉/信任系统

> ⛔ **特殊说明**：镜灵没有"对话信任"的概念——她不说话，所以不存在通过对话建立的信任。她的"信任"等价于**仪式反馈系统**：她对玩家的"信任"是通过符号出现的准确性来表达的。同时，她作为源印载体的特殊身份意味着：她的"信任"直接影响源印的可获取性。

### 4.1 等效信任机制：仪式反馈

```
仪式反馈状态 = 玩家已正确执行的仪式步骤数 (0-6)

  State 0: 无反馈 — 仅○符号（首次接近时浮现）
  State 1-5: 部分反馈 — 累积显示已完成的步骤符号 + 对应的裂纹数
  State 6: 完全反馈 — 玩家面对镜灵按交互键后镜碎触发，通道开启，源印显现
```

不存在"镜灵警觉值"——因为她没有"信任/怀疑"的概念。她只会照出"你是否完成了仪式"——而这不取决于她对你的态度，取决于你做了什么。V1.3中她的反馈可以远程发生，避免玩家每步往返正殿。

### 4.2 对玩家行为的无差别响应

| 行为 | 镜灵的"响应" | 设计说明 |
|------|-------------|---------|
| 正确执行仪式步骤 | 镜面远程浮现对应符号 + 新增一道裂纹 | 她的唯一反馈机制。正面响应不需要语言——符号会说话。 |
| 错误执行仪式步骤 | 无符号，无裂纹 | 沉默 = "不对"。但镜面仍然映出你——她没有"拒绝你"这个功能。 |
| 对她说话 | 镜面继续反射，无文本输出 | 她在倾听——用镜子而非耳朵。 |
| 在她面前做无意义动作 | 镜面继续反射，转头系统不触发额外行为 | 她不判断你的行为是否"有意义"。她只等待仪式步骤的触发条件。 |
| 长时间站在她面前不动 | 镜面继续反射。转头不会自动转回——只要你在触发区内，她就看着你。 | 这是她唯一"主动"的行为：保持注视。最终“启”不再靠静止计时，而是在前五步完成后显示明确交互提示“启镜”。 |
| 触碰镜灵（如果交互系统支持） | 镜面微微震动（极细微的抖动，不是排斥——是"被触碰"这个事件她无法理解——像水面被投石） | 这是她最接近"人类反应"的瞬间——但不是情感反应，是物理/程序性的扰动。 |

## 五、互动风格指南（等效于"对话风格指南"）

> ⛔ **特殊说明**：此NPC无对话。以下为她的**非语言互动风格**——供工程师实现转头/符号/裂纹系统的行为细节参考。

### 5.1 行动基调

- **速度**：转头——15°/s，精确到帧。不是快也不是慢——是"刚好够让你注意到她在动"。不会因为玩家急躁而加速，也不会因为玩家静止而减速。她的速度是常数。
- **节奏**：符号浮现2秒→消失→停顿1秒→转回头。每一步的执行时间精确一致。如果玩家在符号消失前离开触发区——她仍然完成完整的2秒+1秒序列后再转回头（她不会"因为你走了就省掉步骤"）。
- **静止的绝对性**：在没有触发条件的情况下，她是完全的静止。不是"几乎不动"——是"一帧都不动"。她的衣服不会飘（没有风）、她的手指不会抖（没有疲劳）、她的呼吸——不可见（如果有呼吸的话）。她的静止不是人类的静止——是"等待被触发"的设备的静止。
- **裂纹的行为特征**：裂纹的出现没有声音——但有微弱的视觉扰动（镜面上局部亮度闪烁一帧）。裂纹的位置不是随机的——第一条裂纹总是从镜面右上角出现（对应玩家视角的左上），后续裂纹按顺时针方向依次从不同边缘出现。第六条裂纹（开）从中心向外——这是唯一不从边缘开始的裂纹。

### 5.2 标准互动模式（等效于"回应模板"）

**模式：初次识别**
- 触发条件：玩家首次进入3米触发区
- 系统行为：转头（2秒）→ ○浮现2秒 → 停顿1秒 → 转回头
- 叙事意图：○ = "空" / "开始" —— 她确认了你的存在。不是"欢迎"——是"已登记"。这个符号告诉玩家两件事：①她在"看"你；②有一个符号系统——暗示后续步骤会有对应符号。

**模式：步骤验证——正面**
- 触发条件：玩家执行正确仪式步骤
- 系统行为：镜面在正殿远程亮起 → 对应符号浮现2秒 → 新增裂纹；若玩家此时在2米触发区内，则追加转头（2秒）与注视
- 叙事意图：符号 = 验证通过；裂纹 = 进度可见化。玩家通过符号知道"做对了"，通过裂纹知道"还剩几步"。裂纹是仪式倒计时——满六道就碎。

**模式：步骤验证——负面**
- 触发条件：玩家执行错误仪式步骤
- 系统行为：镜面保持反射，无符号，无裂纹；若玩家在触发区内则仅保持正常注视
- 叙事意图：沉默 = 否定。但她不惩罚——她只是不确认。镜面仍映出你——"不对"不等于"拒绝"。她给了你"再试一次"的空间。

**模式：临界——第五步完成后**
- 触发条件：玩家完成第五步"望"后
- 系统行为：○△🔥◇‖↑依次闪现（每个约0.5秒）→ ↑旋转加速；当玩家回到镜灵面前时，她持续注视玩家，并显示“启镜”交互提示
- 叙事意图：符号的连续闪现 = "你做到了——每一步都是对的"。↑旋转 = "最后一步的触发条件已经就绪——你做不做？"她的头不转回——这是她第一次"持续看着你"。她在等你的决定。

**模式：仪式完成——镜碎**
- 触发条件：前五步全部正确后，玩家站在镜灵面前（<1米）并按交互键“启镜”
- 系统行为：↑旋转变白光（1秒）→ 白光扩散（0.5秒）→ 六道裂纹同时扩大（1秒）→ 镜面碎裂（0.5秒——碎片悬浮）→ 碎片映出世界各角落（3秒）→ 碎片聚合为通道（2秒）→ 镜灵面部露出灰白空白
- 叙事意图：这是碎片0003最关键的视觉时刻。镜碎=AI不再需要审视自己=仪式完成了。"开"不是破坏——是释放。碎片映出的六个画面是碎片世界的"记忆闪回"——AI在解散前最后看了她创造的世界一眼。

### 7.2 技术可行性说明：镜面反射Shader

> **风险等级**：P1 — 2D俯视角纸艺风格中实现实时反射+延迟+纸纹理叠加的Shader存在技术不确定性。

**理想方案**（需Spike验证，预估4-6小时）：
- 用Godot的`SubViewport` + `Camera2D`捕获玩家Sprite所在区域
- 环形帧缓冲（3帧@10fps）实现0.3秒延迟
- 叠加纸纹理（粗糙度贴图 + 微淡蓝冷色光晕）
- 凸面镜效果（左右拉伸5%）通过Shader UV偏移实现
- 输出到镜灵镜面的`TextureRect`

**降级方案一**（如实时反射不可行）：
- 预渲染帧序列：根据玩家外观（进入碎片时的外观选择），加载对应的预渲染镜面反射帧序列（6-8帧循环）。放弃实时反射但保留"镜中有倒影"的核心概念。

**降级方案二**（如延迟效果不可行）：
- 关键帧延迟：每隔0.5秒捕获一帧玩家Sprite渲染到镜面贴图。放弃0.3秒延迟但保留反射效果。

**最低降级方案**：
- 静态"水面波纹"Shader（已有古井反射面的技术基础）。玩家靠近时波纹变为模糊的玩家色块轮廓。放弃精确反射，保留"这面镜子在映出什么"的神秘感。
