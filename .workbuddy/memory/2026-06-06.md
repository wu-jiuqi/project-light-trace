## 修复 shuoguang_pull.bat 一键拉取脚本

### 问题诊断
1. **SSL/TLS 连接失败**: Windows Git 默认使用 `schannel` SSL 后端，连接 GitHub 时握手失败
2. **代理未生效**: 系统代理 `127.0.0.1:7897` 已配置但 Git 不会自动读取 HTTP_PROXY 环境变量
3. **分支无上游追踪**: `codex/continue-dev` 分支没有设置 upstream，`git pull origin` 无效
4. **本地未提交变更**: 存在 deleted/untracked 文件可能阻止 pull

### 修复内容
重写了 `C:/Users/lenovo/Desktop/shuoguang_pull.bat`，新增功能:
- 自动切换 SSL 后端为 OpenSSL
- 自动检测/配置 Git 代理 (优先读环境变量，fallback 检测 7897 端口)
- 检测分支上游，无上游时自动指定 `git pull origin <branch>`
- Pull 前自动 stash 本地未提交变更，完成后自动恢复
- 更详细的错误提示和故障排除指引

### 全局 Git 配置变更
- `http.sslBackend` → `openssl`
- `http.proxy` / `https.proxy` → `http://127.0.0.1:7897`

## 二次修复: 本地分支不存在于远程的问题

### 原因
`codex/continue-dev` 是纯本地分支，远程只有 `master` 和 `second`。
脚本用 `git pull origin codex/continue-dev` 时远程找不到 ref，报 `couldn't find remote ref`。

### 修复
- 在 pull 前先用 `git branch -r --list "origin/<branch>"` 检查远程分支是否存在
- 存在 → 正常 `git pull origin <branch>`
- 不存在 → 改用 `git fetch origin` 拉取所有远程追踪，并列出可合并的远程分支

## 三次修复: 添加 fetch 后交互式合并选择

fetch 完成后不再只显示手动命令，而是用 `CHOICE` 命令列出远程分支让用户直接按键选择合并。
- [1] origin/master
- [2] origin/second
- [S] Skip merge (默认 10 秒超时自动跳过)
选中的分支会自动 `git merge` 到当前分支。
