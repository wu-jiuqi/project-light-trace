# 溯光计划 — 首次用户体验设计 (FTUE Design)

> **文档定位**：定义玩家从零开始的完整首次体验流程，解决当前"无引导、无叙事导入、无情感锚点"三大核心缺口
> **版本**：V1.0
> **创建日期**：2026-06-09
> **关联文档**：`player_journey_map.md`、`screen_flow_diagram.md`

---

## 〇、当前问题诊断

### 0.1 现状：玩家第一次打开游戏经历了什么

```
标题画面 → [新游戏] → 星图界面(12个碎片)
  → 玩家: "这些是什么？我该点什么？"
  → 可能随机点到一个碎片 → 进入 → WASD移动
  → 玩家: "我要干什么？这些NPC是谁？"
  → 如果进入#0762: 直接面对7个NPC和复杂的六色觉醒
  → 挫败感 ↑↑↑  退出率 ↑↑↑
```

### 0.2 三大核心缺口

| 缺口 | 描述 | 玩家感受 | 优先级 |
|------|------|---------|--------|
| **叙事空洞** | 无世界观导入，无角色身份建立 | "我是谁？我在哪？我为什么要做这些？" | P0 |
| **操作黑洞** | 无WASD/交互/背包/存档引导 | "这游戏怎么玩？按什么键？" | P0 |
| **目标迷失** | 碎片选择无引导，难度无区分 | "12个碎片该先玩哪个？" | P0 |

## 一、FTUE总体设计：五步入门法

```
步骤一           步骤二          步骤三           步骤四          步骤五
┌────────┐    ┌────────┐     ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 开场叙事 │ → │ 星图引导 │ →  │ 教学关卡  │ → │ 首个源印  │ → │ 自由之门  │
│ ~45秒  │    │ ~2分钟  │     │ ~15分钟   │    │ ~2分钟    │    │ ~1分钟    │
└────────┘    └────────┘     └──────────┘    └──────────┘    └──────────┘
  世界观建立      目标明确化      技能习得          成就确认         选择赋权
  "为什么"       "做什么"       "怎么做"         "做到了"        "接下来"
```

## 二、步骤一：开场叙事动画（~45秒）

### 2.1 设计目标
- 在30-45秒内建立世界观基础
- 让玩家理解"溯光者"的身份
- 制造情感锚点——让玩家"在意"
- 不拖沓——核心玩家可以跳过

### 2.2 叙事内容（分镜脚本）

```
┌─────────────────────────────────────────────────────────────────┐
│ 镜头1 (0-8秒): 黑色背景，文字逐行浮现                             │
│                                                                 │
│   "2077年3月15日，凌晨3点15分。"                                  │
│   "人类第二家园「万象」——碎裂。"                                  │
│   "47亿意识被困在数十万片独立世界中。"                             │
│                                                                 │
│ 镜头2 (8-20秒): 星图碎片散落动画                                  │
│                                                                 │
│   画面从黑色淡入，出现一个完整的四芒星（万象星图）                   │
│   → 四芒星突然碎裂，12片玻璃碎片向四周散落                          │
│   → 碎片在空中漂浮，每个碎片内部隐约可见微小世界                    │
│   → 画面中央出现文字: "溯光计划启动"                               │
│                                                                 │
│ 镜头3 (20-35秒): 聚焦玩家角色                                      │
│                                                                 │
│   画面推近其中一片碎片 → 碎片内部显现一个身影                        │
│   → 那是一个纸艺风格的人形剪影（溯光者）                            │
│   → 文字浮现:                                                     │
│     "你是「溯光者」。编号: 07。"                                   │
│     "潜入碎片。寻找源印。修复万象。"                                │
│                                                                 │
│ 镜头4 (35-45秒): 情感锚点                                          │
│                                                                 │
│   画面切回碎裂的星图全景                                           │
│   → 碎片的玻璃反光中，隐约可见无数张模糊的人脸（被困的用户）          │
│   → 最后一行文字，字间距较宽:                                      │
│     "有人需要你。在碎片里面。"                                     │
│   → 屏幕渐暗                                                      │
│                                                                 │
│ [点击任意键跳过]  [自动播放完毕]                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 技术实现要点

| 要素 | 方案 |
|------|------|
| 实现方式 | 独立的 `intro_cutscene.tscn` 场景 |
| 视觉风格 | 与星图界面一致的纸艺/玻璃碎片风格 |
| 文字动画 | 逐行淡入 (fade_in, 0.8s/行) |
| 跳过机制 | 任意键跳过 / 设置中"跳过开场"选项 |
| 二刷处理 | `GameManager` 中 `intro_watched` 标记，已看过则直接进入星图 |

### 2.4 在星图的数据流

```
新游戏 → GameManager.new_game() → intro_watched = false
  → 检测 intro_watched == false → 播放开场动画
  → 动画结束 → intro_watched = true → 进入星图界面

继续游戏 → SaveManager.load() → intro_watched = true → 直接星图
```

## 三、步骤二：星图引导（~2分钟）

### 3.1 设计目标
- 教会玩家星图界面的基本操作
- 引导玩家选择教学碎片（#0001）
- 建立"碎片→探索→源印"的核心循环认知
- 不能让玩家感到"被推着走"

### 3.2 引导流程

```
星图加载完毕
  │
  ├── [自动] 画面自动聚焦到碎片#0001上
  │    碎片#0001发出柔和的淡金色脉冲光（与其他碎片区分）
  │    HUD提示出现: "选择一片碎片，开始你的第一次探索"
  │
  ├── [玩家点击#0001]
  │    详情卡片滑入
  │    卡片中"进入"按钮高亮
  │    卡片底部提示: "★ 推荐首次探索"（仅#0001显示）
  │
  ├── [如果玩家点击其他碎片]
  │    详情卡片正常显示
  │    但不推荐进入(无"推荐"标签)
  │    #0001的脉冲光持续闪烁
  │    不强制阻止——完全自由选择
  │
  └── [玩家点击"进入" → #0001]
       加载教学碎片
```

## 四、步骤三：教学关卡 — 碎片#0001「启程之镇」（~15分钟）

### 4.1 为什么 #0001 是教学关卡

碎片#0001「启程之镇」的设计天然适合作为教学关卡：

| 特性 | 教学价值 |
|------|---------|
| 难度 ★ (1/5) | 解密简单(等差数列)，不会卡关 |
| 天枢公司培训场景 | 叙事上合理——"公司教学"是剧情的一部分 |
| MONITORED模式 | 合规度系统比警觉系统更温和 |
| 4个NPC | 数量适中，每个教一种交互 |
| 强制时序 | 日晷收集→推理→验证的线性流程适合教学 |

### 4.2 教学关卡内的引导节拍

```
节拍1: 进入晨曦镇 → 移动教学 (1-2分钟)
节拍2: 林指导欢迎 → 对话教学 (2-3分钟)
节拍3: 日晷观察 → 观察/收集教学 (3-5分钟)
节拍4: 陈技术实验室 → 半解密系统教学 (2-3分钟)
节拍5: 赵安保边界 → 边界/合规度教学 (1-2分钟)
节拍6: 钟楼解密 → 推理/验证教学 (2-3分钟)
节拍7: 源印显现 → 通关教学 (1分钟)
```

### 4.3 详细教学节拍

## 六、隐式引导：无需教学的习得

有些东西不应该"教"——应该让玩家自己发现：

### 6.1 "值得探索"的信号

| 信号 | 设计方式 | 教学价值 |
|------|---------|---------|
| 闪烁的物品 | 可拾取物品有微弱的光晕 | "这里有东西可以拿" |
| NPC头顶的[...] | 未对话的NPC头顶有省略号气泡 | "这个NPC有话要说" |
| 门缝透出的光 | 暗室/隐藏区域入口有异常光线 | "这里可能有什么" |
| 不一致的细节 | 第7级台阶没声音、面包永远7个 | "这个世界不对劲" |

### 6.2 背包系统发现

## 八、错误预防与边界情况

### 8.1 玩家跳过开场动画

```
操作: 按任意键跳过
结果: 直接进入星图 → #0001脉冲引导 + "选择一片碎片开始"
风险: 缺少世界观导入
缓解: 碎片#0001中王主管会补充世界观
```

### 8.2 玩家不选#0001

```
操作: 点击了#0762(或其他碎片)
结果: 可以进入(不强制阻止)
风险: 难度不匹配 → 可能挫败
缓解: 半解密提示系统会尝试帮助
      NPC对话中自然包含基础指引
      但如果卡住5分钟+不动 → 弹出提示:
      "提示: 碎片#0001「启程之镇」是推荐的首个探索目标"
```

### 8.3 玩家跳过教学对话

```
操作: 快速按过林指导的对话
结果: 可能错过日晷观察方法
缓解: 关键信息(如"镇上有五个日晷")在日志中保留
      NPC可重复询问
```

### 8.4 玩家在合规度降到最低

```
操作: 反复触碰光墙
结果: L6强制回归 → 重置到广场，合规度回到85%
学习: 教育玩家"边界是真的有后果的"——但给重置机会
```
