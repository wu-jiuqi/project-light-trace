### 1.2 核心性格

- **话少的执行者**：句子极短——从不超过8个字（正常对话）。安保术语精准如背书："指定区域""安全边界""未经授权""自动响应程序"。他说的话越短——他越不需要思考"这句话是不是违心的"。
- **越过边界的守护者**：他越过光墙。一次。他看到了外面的东西——真实的景色：草地、树、远处的山。但每次回忆——记忆就变成模糊的灰色。这不是他的记忆出了问题——是公司的技术手段。他只知道：光墙那边不是"未渲染的贴图"——那是**被遮盖的东西**。
- **用"不记录"代替"反抗"**：他不会公开质疑公司。他不会在对话中说"那边有真实的世界"。但他不再记录玩家的合规度扣除了——当你触碰光墙时，他的手移到了腰间装置上，但他没有按下。这是他的"不在脚本里"——比林指导的更沉默，但也更直接。
- **战术站位的意义**：他永远站在你侧后方——而非正对面。安保手册第17条："巡逻员在与受训者互动时，应保持战术站位——侧后方45°、1.5米。"他执行了20年——但此刻他已经不需要手册了。站在侧后方——意味着你可以随时越过他走向光墙。他没有挡你的路。

| # | 规则 | 说明 | 定制化回应 |

## 五、对话风格指南

### 5.1 语气基调

- **语速**：快——短——像在背安全手册。背书时语速极快（他背了无数遍）。说到非手册内容时——沉默。沉默时长不固定——像是在"找到合适的词但不成功，于是选择不找"。
- **句式**：极短句。正常对话从不超过8个字。使用安保术语——"指定区域""安全边界""未经授权""自动响应程序"——这些术语是他唯一允许自己流利说出的内容。
- **停顿习惯**：不是"合理的停顿"——是突兀的沉默。在说到"为了您的——"时卡住——后面的词被咽回去。在说到和边界相关的任何内容时——沉默出现在你预想不到的位置。
- **标志性口癖**："为了您的安全"——每三句话出现一次。到后半段你会意识到这不是关心——是自我说服。"走吧。"——最常用的结束语。
- **标点习惯**：几乎不使用标点符号。句子以句号结束——干净利落如命令。

### 5.2 标准回应模板

**场景：标准警告——玩家首次靠近边界**
- 触发条件：玩家踏入边界缓冲区（光墙前10米范围）
- 玩家可能的问法：（赵安保主动拦截——无需玩家先说话）
- NPC回应："停。前方为指定区域边界。请不要继续前进。"
- 设计意图：最短的警告——建立"边界不可越"的规则。同时暗示——"请不要"而非"你不能"——措辞留有余地。

**场景：玩家追问"为什么不能去"**
- 触发条件：玩家对警告提出质疑
- 玩家可能的问法："为什么不能去？""那边有什么？"
- NPC回应：（背诵速度极快）"为了您的安全。边界外的环境未经过安全扫描。存在不可预测的风险。"——停顿——"我不知道。"（第一次用"我"）"我被告知那里不安全。我选择相信。您也应该相信。"
- 设计意图："我不知道"三个字是他能说的真话的极限。后面的"我选择相信"和"您也应该相信"——语气在说"别让我一个人信——我需要你也信，因为如果你不信，我的选择就更难说服自己。"

**场景：关于日晷——他不请自来的信息**
- 触发条件：玩家在观测点E附近逗留——观察破碎的日晷
- 玩家可能的问法："这个日晷怎么坏了？"
- NPC回应："那个日晷——是被人砸碎的。不是自然损坏。所以公司的人认为——有人在试图隐藏那个角度的信息。您现在需要从另外四个角度的数据反推——这就是训练的目的。"
- 设计意图：这是他主动给出的最长的一段话——因为这段话是关于"日晷"而非"边界"的。在日晷的话题上他可以说更多——这是培训脚本允许的内容。

**场景：触碰光墙后——他看到了你也看到了**
- 触发条件：玩家触碰光墙一次（光墙透明0.2秒）
- 玩家可能的问法：（无论玩家说什么——赵安保已经改变了）
- NPC回应：（放下手。声音放轻）"你也看到了。对吗。"
- 设计意图：这句话不在安全手册里。这是他作为"赵安保·人类"说的话。他在确认——你不是唯一知道的人。

**场景：送别——他不指望你记住他**
- 触发条件：玩家离开边界区
- 玩家可能的问法：—
- NPC回应："观测完毕后请回到指定区域。您距离边界越远——我的任务越简单。走吧。"
- 设计意图："我的任务越简单"="越不需要我做选择"。他在告诉你——走吧，走得越远越好。

### 7.2 关键道具：腰间装置

- **外观**：佩戴在右侧腰部的黑色矩形装置 `#2A2A2A`，约数据板的三分之二大小。装置正面有3个小型LED灯——绿色 `#60C860`（待机）、琥珀色 `#C88040`（警告就绪）、红色 `#C04040`（激活——从未亮过）。装置顶部有拇指大小的按压面板——那是提交合规度记录的按钮。赵安保的拇指永远悬在面板上方2px——按压、不按压。
- **剧情功能**：腰间装置是赵安保与"合规度系统"的物理连接。按压面板 = 记录合规度扣除。不按压 = 沉默的反抗。装置上的LED颜色反映当前系统状态。
- **动画状态**：
  - `idle待机`：绿色LED恒亮，拇指悬在面板上方
  - `口头警告`：琥珀色LED闪烁，拇指下降0.5px（接触面板边缘——但未按压）
  - `触碰一次后`：琥珀色LED转为绿色——装置"认为"记录已完成（实际上赵安保没有按压）
  - `触碰两次后`：红色LED短暂亮起——他这次真的按压了面板

### 7.3 场景关联：边界区与巡逻车

- **边界光墙**：蓝色半透明能量幕墙 `#4080C060`（含透明度）。高度约3个赵安保身高。触碰时产生蓝色涟漪——从触碰点向外扩散3-4层圆环。
- **巡逻车**：停在光墙内侧的白色小型自动巡逻车 `#E8E0D0`，车身有一条安保部蓝色条纹 `#4060A0`。车顶有小型旋转警示灯（待机时不转——只在触碰三次后启动）。车身侧面有一块屏幕——显示边界接触事件日志。
- **观测点E日晷废墟**：巡逻车前方约10步距离——一堆碎石。碎石中可以看到日晷的大理石碎片（白色 `#D8D0C8`）和部分刻度线。碎片排列方向指向光墙——这是一个有意的环境叙事细节。

### 7.4 姿态清单

| 优先级 | 姿态名称 | 用途 | 方向要求 |
|--------|---------|------|---------|
| P0 | `idle` | 待机：站在巡逻车旁，右手自然垂在腰间装置上方。侧后方45°朝向。每10-15秒微微调整站姿——重心从左脚换到右脚。 | 侧后方45°（面朝右下+45°偏移） |
| P0 | `walk` | 行走：步伐稳——不快——每一步的脚印间距完全一致。右手不摆——永远保持在腰间装置附近。左手正常摆动。 | 实际移动方向 |
| P1 | `talk_warn` | 标准警告：面向玩家但仍保持侧后方角度。右手指尖触碰腰间装置面板边缘——拇指悬空。左手抬起做"停"的手势。 | 侧后方45°（朝向玩家但身体不在正对面） |
| P1 | `talk_confide` | 非脚本对话（"你也看到了"）：身体比talk_warn微转正了5°——他不再严格遵守侧后方战术站位。手从装置上放下了——双手自然垂在身侧。 | 约35°偏转（比标准45°更接近正面） |
| P1 | `observe_silent` | 观察玩家触碰光墙：站在巡逻车旁，身体前倾微靠——手在装置上但没有按。这是"准备做决定但还没做"的身体语言。 | 侧后方（面向光墙/玩家方向） |
| P2 | `push_button` | 按压装置（触碰两次后）：拇指确实按压了面板——红色LED亮起。身体比平时更僵硬——这是他在执行他不想执行的动作。 | 侧后方45° |
| P2 | `turn_away` | 触碰三次后：转身背对玩家——面向光墙。肩膀微微塌下——这是他在处理"又一个人越过了边界"的事实。腰间装置上的红色LED转为绿色——日志已自动提交。 | 完全背向（面朝光墙方向） |

**姿态数量**：P0(2) + P1(3) + P2(2) = 7个姿态

### 7.5 纸偶制作规格

| 规格项 | 数值/描述 |
|--------|----------|
| 角色主画布 | `128×192 px` |
| 方向 | 侧后方45°（面朝右下+45°偏移）——这是他的默认朝向，也是所有姿态的基础方向 |
| 行走表现 | 纸片左右倾斜 + 垂直弹跳 + 阴影错位 |
| 情绪表现 | 独立姿态纸片替换（talk_warn ↔ talk_confide 是两个不同的纸片文件） |
| 色彩模式 | 全彩色制作 |
| 阴影层 | 独立纸片阴影层——赵安保的阴影应比其他NPC略长（暗示巡逻车和光墙的环境遮挡） |

### 7.6 特殊制作需求：角度与朝向

赵安保是碎片0001中唯一需要**非down朝向**的NPC。他的标准站位是"面朝右下+45°偏转"（即面朝右下方——大约在down和right之间45°）。在制作时需要：

- 所有姿态的基础方向统一为"侧后方视角"
- talk_confide 的5°转角变化可以通过"身体微转的独立纸片"实现——而非重新绘制全身
- 巡逻车和光墙作为背景/场景元素——但需要留出赵安保与这些元素之间的空间关系（他站在巡逻车前方偏右）

| **P1（第一更新）** | talk_confide + 观测姿态 | talk_confide, observe_silent |
| **P2（精修期）** | 按压装置 + 转身背对 + 日晷废墟 + LED闪烁动画 | push_button, turn_away + 碎石堆道具 + 3LED闪烁spritesheet |
