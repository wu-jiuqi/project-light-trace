# 溯光计划 UI 面板产品需求文档（简单 PRD）

**版本**: v0.1  
**作者**: 产品经理 许清楚  
**日期**: 2025-07-11  
**语言**: 中文  
**目标引擎**: Godot 4.x (GDScript)  
**分辨率基准**: 1280×720

---

## 总则：纸雕风格设计约束

所有 4 个面板必须遵循 DialogueBox.tscn 已确立的纸雕暗黑童话 UI 风格：

- **色调**: 暗黄/棕色古旧纸张氛围，暗色半透明遮罩层
- **字体色**: 正文 `#2C1F14` (暗棕)，标签 `#B28E51` (古铜金)
- **按钮体系**: 复用 `button_plate.png` / `button_plate_alt.png` 的 StyleBoxTexture
- **底衬**: 统一使用 `large_page.png` 作为大页纸背景层
- **装饰**: 角落指南针、横幅、山峦等纸雕装饰可按需裁量使用
- **光源**: 左上投射，投影向右下
- **安全区**: 所有文字区域避免与可读字符重叠，运行时渲染

## 面板一：Backpack（背包面板）

### 1. 项目信息

| 字段 | 值 |
|------|-----|
| Project Name | `backpack_panel` |
| Scene Path | `scenes/ui/Backpack.tscn` |
| Script Path | `scripts/ui/backpack.gd` |

### 2. 产品定义

**产品目标**:
1. 让玩家能够快速浏览和识别当前携带的所有物品
2. 提供物品详情查看能力，强化收集反馈
3. 与对话系统无缝衔接，可在对话中随时打开背包进行"给予"操作

**用户故事**:
- As a 玩家, I want 打开背包时看到物品以网格形式排列 with 图标和名称，so that 我能快速找到目标物品
- As a 玩家, I want 点击某个物品后查看它的详细描述，so that 我了解它的用途和来源
- As a 玩家, I want 从背包中选择物品给予 NPC，so that 我能推进剧情发展
- As a 玩家, I want 按关闭按钮或 ESC 键退出背包，so that 我能顺畅返回游戏/对话
- As a 玩家, I want 看到物品数量不足时的灰态提示，so that 我知道哪些物品即将耗尽

| BP-04 | P1 | 物品详情面板：点击物品后右侧显示描述文本 |
| BP-05 | P1 | 物品数量角标显示 |

## 面板二：DialogueHistory（对话历史面板）

### 1. 项目信息

| 字段 | 值 |
|------|-----|
| Project Name | `dialogue_history_panel` |
| Scene Path | `scenes/ui/DialogueHistory.tscn` |
| Script Path | `scripts/ui/dialogue_history.gd` |

### 2. 产品定义

**产品目标**:
1. 让玩家回看已发生的所有对话，避免错过关键信息
2. 提供流畅的滚动浏览体验，适配长对话场景
3. 与 DialogueBox 视觉风格一致，让玩家感觉是同一本书的不同页码

**用户故事**:
- As a 玩家, I want 打开对话历史时看到从对话开始到当前的所有记录，so that 我能回顾错过的剧情线索
- As a 玩家, I want 使用滚轮/拖拽滚动浏览历史，so that 我能快速定位某段对话
- As a 玩家, I want 对话记录区分 NPC 发言和玩家发言（不同的对齐/颜色），so that 我能分辨发言人
- As a 玩家, I want 按返回按钮关闭历史面板，so that 我能继续当前对话
- As a 玩家, I want 在历史面板中看到对话选项的选择结果，so that 我了解之前的抉择

### 3. 技术规范

**需求池**:

| ID | 优先级 | 需求描述 |
|----|--------|----------|
| DH-01 | P0 | ScrollContainer 内 RichTextLabel，显示完整对话记录 |
| DH-02 | P0 | 返回/关闭按钮 |
| DH-03 | P0 | NPC 发言（左对齐/米色底）与玩家发言（右对齐/浅棕底）视觉区分 |
| DH-04 | P1 | 自动滚动到最新一条（scroll_following） |
| DH-05 | P1 | 发言者名称前缀显示 |
| DH-06 | P1 | 玩家选择的选项以斜体/特殊色标注 |
| DH-07 | P2 | 关键词高亮（线索词以金色标记） |
| DH-08 | P2 | 对话时间戳显示 |
| DH-09 | P2 | 从历史面板直接跳转到某段对话的上下文（仅限当前场景记忆范围内） |

### 2. 产品定义

**产品目标**:
1. 提供线索/证据的集中查看界面，强化侦探叙事体验
2. 展示线索之间的关联关系，辅助玩家推理
3. 每个线索可点击展开详情，营造"翻阅档案"的沉浸感

**用户故事**:
- As a 玩家, I want 看到已收集的线索以卡片形式排列，so that 我对调查进度有整体认知

- As a 玩家, I want 按关闭按钮退出线索面板，so that 我返回游戏

### 2. 产品定义

**产品目标**:
1. 提供游戏核心设置的调整入口，覆盖音量、画面、语言等基本项
2. 保持与游戏整体纸雕风格的一致性，设置面板不跳出沉浸感

| SP-04 | P0 | 关闭按钮（等同于取消+关闭） |
