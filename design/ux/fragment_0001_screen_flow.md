## 一、碎片内部界面层次模型

碎片0001的界面分为三个覆盖层级，叠加于游戏视口（Camera2D）之上：

```
┌──────────────────────────────────────────────────────────────────┐
│               L0: 全局覆盖层 (CanvasLayer 128+)                   │
│  暂停菜单(PauseMenu) | 背包UI(BackpackUI) | 对话面板(ChatDialogue) │
│  胜利画面(VictoryScreen) | SceneFader 黑屏过渡                    │
├──────────────────────────────────────────────────────────────────┤
│               L1: 碎片HUD层 (CanvasLayer 程序化生成)              │
│  合规度条 | 交互提示 | 进度追踪器 | 角度面板 | 消息弹窗           │
├──────────────────────────────────────────────────────────────────┤
│               L0-: 游戏视口 (Camera2D, zoom=1.9)                  │
│  玩家 | NPCx4 | 日晷x5 | 建筑群 | 光墙 | 钟楼暗门 | 雾效        │
└──────────────────────────────────────────────────────────────────┘
```

**层级规则**：
- L1 HUD 元素由 `fragment_0001.gd` 的 `_ready()` 程序化创建为 CanvasLayer 子节点，z-index 范围 10-50
- L0 全局覆盖层由 Autoload 管理（PauseMenu=100, ChatDialogue=128, BackpackUI=127），叠加于碎片场景之上
- 碎片场景本身不包含任何 Autoload 级覆盖层节点——全部通过 Autoload 或程序化 CanvasLayer 实现

## 三、HUD层详细设计

HUD层由 `fragment_0001.gd` 在 `_ready()` 中程序化创建，使用 CanvasLayer 节点。**所有HUD元素均为非模态**——不阻挡玩家移动。

### 3.1 交互提示 (InteractHint)

```
┌──────────────────────────────────┐
│   [E] 观测日晷                    │  ← 半透明深底白字，位于画面底部中央
└──────────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 触发条件 | 玩家进入日晷/NPC/可交互物的 Area2D 范围内 |
| 显示位置 | 画面底部中央，player上方约40px |
| 可见性 | 靠近时淡入0.15s，离开时淡出0.1s |
| 文本内容 | 根据交互目标动态变化 |
| 样式 | 半透明深色底(alpha=0.75)，白色文字，8px圆角 |

**文本变体**：
| 交互目标 | 提示文本 |
|---------|---------|
| 日晷A/B/C/D | `[E] 观测日晷` |
| 日晷E（已碎） | `[E] 查看残骸` |
| 林指导 | `[E] 与林指导对话` |
| 陈技术 | `[E] 与陈技术对话` |
| 王主管 | `[E] 与王主管对话` |
| 赵安保 | `[E] 与赵安保对话` |
| 钟楼暗门（解锁后） | `[E] 进入暗室` |
| 钟楼日晷B（全部观测后） | `[E] 校准日晷指针` |
| 暗室内源印 | `[E] 触碰晨曦之印` |

### 3.2 合规度条 (ComplianceBar)

```
┌─────────────────────────────────────────────────────┐
│  合规度  ████████████████████████████████░░░░  85%   │  ← 蓝色条 + 百分比
└─────────────────────────────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 始终可见 | 是（MONITORED 模式独有UI，后续碎片使用不同的警觉条样式） |
| 显示位置 | 画面右上角 |
| 初始值 | 100% |
| 颜色方案 | 蓝色基调（与后续碎片的红色警觉条区分） |
| 满条颜色 | `#4A90D9`（天枢蓝） |
| 警告颜色（<70%） | `#E8A838`（琥珀色闪烁） |
| 危险颜色（<40%） | `#D94A4A`（红色脉冲） |
| 条宽 | 200px，高16px |
| 动画 | 扣除时平滑缩短(0.3s EASE)，有短暂红色闪光 |
| 文字 | 右侧百分比数字 + 前方"合规度"标签 |

### 3.3 角度面板 (AnglePanel)

```
┌──────────────┐
│   当前角度    │
│     XX°      │  ← 仅在钟楼校准模式时可见
└──────────────┘
```

| 属性 | 值 |
|------|-----|
| 触发条件 | 玩家进入钟楼日晷B的校准交互状态 |
| 显示位置 | 画面中央偏上 |
| 可见性 | 仅在主动旋转日晷指针时显示 |
| 反馈变化 | 普通值→白色；60°→橙色("接近了"); 66°→金色("正确") |
| 消失条件 | 退出校准交互(Esc / 走远) 或 校准成功 |

### 3.4 消息弹窗 (MessagePanel)

```
┌──────────────────────────────────────────────┐
│  [林指导]                                    │  ← 发送者名称 (淡金)
│                                              │
│  溯光者，编号确认通过。欢迎来到溯光计划        │
```

| 属性 | 值 |

| 显示位置 | 画面顶部，从上方滑入 |

## 十、输入优先级（碎片内）

```
L0 (最高): 系统级 — ESC暂停菜单 / Alt+F4
L1:        覆盖层 — 对话面板内输入 / 胜利画面按钮 / 合规度强制回归(不可中断)
L2:        子面板 — 线索面板内键盘 / 校准面板按钮 / 便笺阅读面板
L3:        半模态 — 背包UI (B键开关)
L4 (最低): 游戏视口 — WASD 移动 / E 交互 / Tab 线索面板 / ←→ 校准日晷
```

**特殊规则**：
- 林指导幕间事件期间：**封锁所有输入**（不可跳过，不可中断）
- 合规度强制回归期间：**封锁所有输入**（自动播放完成）
- 钟楼校准模式：锁定WASD移动，但保留←→和Esc

## 十二、与全局流转的对接点

碎片0001的界面流转在以下节点与全局 `screen_flow_diagram.md` 对接：

| 本碎片节点 | 全局流转对应 | 对接方式 |
|-----------|------------|---------|
| 进入碎片 | 星图→解密过渡→碎片世界 | FragmentManager.enter_fragment() |
| ESC暂停菜单 | L0覆盖层·暂停菜单 | 全局PauseMenu Autoload接管 |
| B键背包 | L0覆盖层·背包UI | 全局BackpackUI Autoload接管 |
| NPC对话面板 | L2子界面·对话面板 | 全局ChatDialogue Autoload接管 |
| 胜利画面 | 全局·胜利画面 | FragmentManager.complete_fragment() → 跳转星图 |
| 返回星图 | 星图界面 | SceneManager.change_scene("star_map") |

碎片0001**不需要**重新实现这些全局覆盖层——它们由Autoload统一管理，自动叠加在碎片场景之上。
