## 维度1：解密逻辑合理性

### 1.1 核心解密逻辑逐步推演

以玩家视角代入完整推理流程，用四条线索的实际数据进行交叉验证：

```
线索一：石灯笼和歌（12字）
  完整和歌：月読水清 / 三灯玉井 / 掌声天開
  意象递进：月→水→灯→玉→掌→天
  
  解读映射：
    月読水清 → 月光照水，水面清澈 → 祓（净手）
    三灯     → 第三盏灯         → 祝（点第三灯）
    玉井     → 勾玉和井         → 捧（向井投玉）
    掌声     → 击掌的声音       → 祈（击掌）
    天開     → 天空开裂         → 待（指天）→开（通道打开）

  和歌自然意象递进 = 正确仪式顺序：祓→祝→捧→祈→待→开

线索二：神主念诵（排除错误相邻关系）
  循环1后："祓和捧——不应该相邻。"
  循环2后："祈和开——顺序不可能。祈必须在开之前很久。"
  循环3后："捧和待——中间缺了一步。"
  
  排除：祓-捧相邻、祈-开相邻、捧-待相邻
  → 从剩余排列空间中筛选，与和歌顺序交叉验证

线索三：枯井月影 + 宝物殿勾玉盒
  井水颜色循环：红→蓝→黄→绿→紫→白
  勾玉盒盖内部对应表：青=祓、緑=祝、紫=捧、赤=祈、黄=待、白=開
  
  颜色-步骤对应：蓝→祓、绿→祝、紫→捧、红→祈、黄→待、白→开
  井水颜色出现顺序：红(祈/4)→蓝(祓/1)→黄(待/5)→绿(祝/2)→紫(捧/3)→白(开/6)
  
  → 颜色顺序 ≠ 仪式顺序（这是故意设计的"误导"——玩家需要认识到井水颜色顺序是乱序，需要配合勾玉盒重排）

线索四：巫女·镜（逐步验证）
  每完成一步正确步骤 → 走近巫女 → 镜面浮现对应符号
  ○(开始) → △(祓) → 🔥(祝) → ◇(捧) → ||(祈) → ↑(待) → ✦(开)
  
  错误步骤 → 无符号反馈
```

**最终交叉验证**：

| 验证维度 | 推导结果 | 结论 |
|---------|---------|------|
| 和歌意象递进 | 祓→祝→捧→祈→待→开 | ✅ 主线索，给出完整顺序 |
| 神主排除相邻 | 排除(祓-捧)(祈-开)(捧-待) | ✅ 与和歌顺序无矛盾 |
| 井水颜色+勾玉盒 | 颜色→步骤映射表 | ✅ 提供验证依据 |
| 巫女符号反馈 | 每步执行后镜面验证 | ✅ 逐步骤确认 |

**结论**：四条线索的交叉验证体系完整，逻辑链条闭环。和歌作为主线索给出完整顺序，神主念诵排除错误组合，井水颜色提供映射验证，巫女提供逐步骤确认——四者构成"推导→排除→验证→确认"的完整认知路径。

| 审查项 | 结论 | 详情 |
|--------|------|------|

### 1.3 发现的问题

**问题 #1（P1）：和歌给出完整顺序，导致神主念诵和井水月影的推理价值被稀释**

- **位置**：§四.2 线索二（石灯笼和歌）+ §四.3 解密步骤详解
- **问题**：当前设计中，12字和歌的意象递进（月→水→灯→玉→掌→天）直接映射为完整仪式顺序（祓→祝→捧→祈→待→开）。一个收集完全部12字的玩家，几乎不需要神主念诵和井水月影就能直接获得正确答案。这使得四条线索中三条线索的"推理功能"集中到了和歌上，另外两条线索退化为"可选验证"。
- **影响**：对于★★难度的碎片，这并非致命问题——但降低了"多线索交叉验证"这一核心教学目的（教会玩家多信息源整合）的实现效果。当一个线索就能给出完整答案时，"交叉验证"变成了"交叉确认"。
- **建议**：考虑将和歌的信息拆散——让和歌的词序本身也是"乱序"的，12字按当前石灯笼位置排列无法直接组成通顺和歌。玩家需要从井水颜色循环或神主念诵中获得"排序键"来重新排列和歌。例如：井水颜色出现的顺序（红→蓝→黄→绿→紫→白）对应的步骤序号（4→1→5→2→3→6）可以作为和歌的重排键。这样三条线索的"推理贡献"更均衡。
- **严重度**：P1

**问题 #2（P2）：仅依赖神主念诵排除相邻关系时，收敛速度偏慢**

- **位置**：§四.2 线索一（神主念诵）+ §四.3 步骤二
- **问题**：当前设计中，神主每5分钟一个循环，每次循环后指出一对错误相邻关系。要获得3-4对排除关系，玩家需要听3-4个循环 = 15-20分钟。如果玩家没有同时推进和歌收集和井水观察，纯靠神主线索的进度会很慢——可能导致部分玩家在"等待下一个循环"中流失。
- **建议**：增加一个加速机制——当玩家向神主询问"还有什么不对的吗？"时，神主会额外说出一对排除关系（每次询问限一次，总共最多额外2次）。或者将念诵循环缩短至3分钟一次。
- **严重度**：P2

**问题 #3（P2）：击掌21次的"和歌总音节数"计算逻辑对中文玩家不直观**

- **位置**：§四.3 步骤三（祈步骤）+ §十二.3 玩家体验测试
- **问题**：击掌次数=和歌总音节数=21次。这个计算的来源是日语和歌的音节分割（月=2、読=2、水=2、清=2…），但游戏的主要语言是中文。中文玩家不会天然地用日语音节来读这些汉字，导致"21"这个数字的来源不够透明。设计文档本身也意识到了这个问题（§十二.3中提问"是否过于繁琐"）。
- **建议**：①在参道石碑（鸟居旁）上直接刻出"二十一"这个数字，作为明面提示；②或者在"祈"步骤的提示中将"21次击掌"作为一个"发现"而非"计算"——由神主的念诵中偶尔混入数字词（"二十一の祈り"）来暗示。降低计算门槛，保留仪式感。
- **严重度**：P2

### 2.5 发现的问题

**问题 #4（P2）：0002的"被留下的人"在0003中缺少呼应**

- **位置**：§八.2 玩法-叙事独特性
- **问题**：碎片0002的核心情感重量来自"你选择留下谁"。但碎片0003的设计中，完全没有提到0002留下的那个人对这个碎片的影响。虽然设计文档在§十三中提到"被留下的人"会在碎片3333中以回声形式出现，但在0003这个紧接0002的碎片中没有任何回应——可能让玩家觉得"我当初的选择不重要"。
- **建议**：在0003的某个环境叙事节拍中，加入一个微弱的呼应——例如：枯井多倒影中，有一个倒影的轮廓隐约像是0002中被留下的那个NPC；或者神主在节点2的念诵中混入一个和"离别"相关的词（"別れ——給い——"），暗示AI记得你的上一个选择。

### 3.4 发现的问题

**问题 #5（P2）：暗线B勾玉盒底刻字的信息密度对"种子"阶段偏高**

### 4.5 发现的问题

**问题 #7（P1）：巫女镜面反射+0.3秒延迟的Shader实现存在技术不确定性**

- **位置**：§三.3 巫女·镜面部设计
- **问题**：在2D俯视角纸艺风格的Godot项目中，实现一面"实时反射玩家形象+0.3秒延迟+纸纹理叠加"的镜子，需要：①将玩家Sprite渲染到RenderTexture；②对RenderTexture做0.3秒延迟缓冲（环形帧缓冲）；③叠加纸纹理。步骤②在Godot 4.x中需要额外的帧缓冲管理逻辑，不是开箱即用的功能。当前项目的Shader复杂度主要在纸艺风格渲染（多层半透明叠加），尚未涉及实时反射管线。
- **建议**：①提前做一个技术验证原型（spike），用最简单的RenderTexture+延迟缓冲验证可行性；②如果延迟反射不可行，降级方案：镜面使用预渲染的"反射"贴图（根据玩家进入碎片时选择的外观，加载对应的预渲染镜面反射帧序列），放弃实时反射但保留延迟效果。
- **严重度**：P1

**问题 #8（P2）：镜中通道的7块碎片"各自映出不同画面"的视觉实现可能超出纸艺风格约束**

- **位置**：§七.2 环境叙事节拍#6 + §九.3 镜中通道视觉设计
- **问题**：镜中通道被定位为"非纸艺——超现实的碎片悬浮空间。碎片是真实的镜面碎片（shader特效）"。这与整体纸艺视觉风格存在风格断裂风险——玩家在前6个区域一直沉浸在纸模型/微缩景观的视觉语言中，突然进入一个"超现实Shader空间"可能打破沉浸感。
- **建议**：将镜中通道也纳入纸艺风格系统——碎片不是"真实的镜面碎片"，而是"纸做的镜面碎片"。具体方案：①碎片边缘保留撕纸质感（不规则毛边）；②碎片上的画面不是"视频"，而是"纸上的画"（静态但微微飘动）；③碎片悬浮的物理效果模拟"纸片在空气中飘浮"而非"镜片悬浮"。
- **严重度**：P2

**问题 #9（P2）：7个区域的纸艺美术资产总量需要评估产能**

- **位置**：§九 视觉与美术方向概要
- **问题**：碎片0003有7个独立区域，每个区域需要纸艺建筑外立面、地面铺装、室内陈设、道具等。对标0762的6个房间，0003多一个区域（镜中通道），且鸟居、枯井、宝物殿、本殿的建筑复杂度高于0762的大部分房间（花店、面包店等）。建议在进入生产前做一个粗略的美术资产量估算。
- **建议**：在步骤7（物品设计）完成后，输出一份物品数量统计，与0762的物品数量做对比，评估产能是否匹配。
- **严重度**：P2

### 7.5 发现的问题

**问题 #12（P1）：警觉值在高区间缺乏降警觉手段**

- **位置**：§六.3 警觉值变化规则 + §七.1 软锁风险评估
- **问题**：当前设计中，警觉值**只能通过正确执行仪式步骤（每次-10%）和收集完整和歌（一次性-5%）来降低。如果玩家在仪式早期（完成0-1步）时就触发了较高的警觉值（例如砸碎石灯笼+15%、反复无效交互累计+25%），而和歌又未收集完整，那么降警觉的手段非常有限。在L4-L5区间（51-90%）可能陷入"不敢再试错但也不知道正确步骤"的僵局。
- **建议**：增加一个降警觉的环境交互——例如"在手水舍净手（即使不作为仪式步骤，仅作为自由交互）每次-3%警觉"，通过"仪式性净化"的叙事逻辑合理化降警觉。或者：当玩家在宝物殿阅读卷轴时，警觉值缓慢下降（-1%/秒，最多-10%）——因为"你在学习仪式的规则"。
- **严重度**：P1

**问题 #13（P2）：击掌21次的操作体验可能引发疲劳感**

- **位置**：§四.3 步骤三（祈）+ §十二.3 玩家体验测试
- **问题**：21次连续击掌（按2-2-2-2-1-2-2-1-2-2-1-2节奏）在纸面上是一个优雅的设计（和歌总音节数=21）。但在实际游玩中，连续21次点击同一个交互区域可能让部分玩家感到繁琐——特别是当他们已经知道正确次数后，重复执行缺乏新鲜感。设计文档本身的测试要点（§十二.3）也提出了这一问题。
- **建议**：①将21次改为"按节奏击掌"（玩家按住交互键→角色按预设节奏自动击掌21次→玩家只需在正确的起始时刻按下并保持，而非21次独立点击）；②或在首次完成后，后续重试（如因前序步骤错误需要重做）缩短为"快速击掌3次"作为简化版。③如果保留21次独立点击，需要确保点击响应灵敏且容错窗口宽（±0.3秒）。
- **严重度**：P2

## 总结评价

碎片0003「月下神社」是一份**质量极高的关卡设计文档**。它的核心优势在于三点：

**第一，叙事与玩法的深度融合**。这不仅仅是一个"在神社里解谜"的碎片——解密本身就是叙事。"替AI完成她无法完成的仪式"这个核心概念同时驱动了解密逻辑（六步净化仪式）、NPC设计（神主=指令指针、巫女=验证模块）、和情感弧线（从困惑到庄严到释然）。仪式步骤的物理交互（舀水、点灯、投玉、击掌、仰望、站立）让玩家用身体参与AI的自我修复——这是"玩法即叙事"的教科书级示范。

**第二，NPC设计的"非人"美学**。神主的循环念诵+碎片化语言、巫女的镜面面部+沉默反馈——两个NPC都精准地传达了"他们是AI的功能部件，而非人类"的设定。神主在仪式完成时第一次说出完整、连贯的句子——这个叙事节拍的冲击力来自前面所有的碎片化语言的铺垫。巫女的"从不说话+镜面符号验证"创造了独特的互动模式——她不告诉你答案，但她告诉你"你对了"。

**第三，暗线植入的层次感和精准度**。强制触发的暗线A核心推进（神主独白）找准了情感高潮的时机；可选触发的暗线B种子（勾玉盒底刻字）内容扎实但不会喧宾夺主；极隐晦的暗线C种子（0.3秒镜异影+枯井倒影）为后续碎片播种了恰到好处的好奇心。更难得的是——三条暗线的植入都**不打断**当前的仪式叙事节奏。

**核心风险**：解密设计中"和歌直接给出完整顺序"导致的多线索推理价值不均衡（P1），以及巫女镜面反射Shader的技术可行性（P1），是需要在步骤3中重点修复的两个问题。前者影响"多信息源交叉验证"的教学目的达成度，后者影响核心视觉锚点的实现。

综合来看：**🟡 基本通过** — 0个P0阻塞问题，3个P1需修复，10个P2优化建议。设计文档可以在修复P1问题后进入步骤4（NPC设计）。
