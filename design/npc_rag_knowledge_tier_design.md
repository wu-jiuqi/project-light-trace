# NPC RAG 知识分层设计原则

> 溯光计划 | 全局设计规范  
> 版本：V1.0 | 2026-06-10  
> 状态：生效中 — 所有碎片NPC RAG知识库设计必须遵守此规范

---

## 零、为什么需要这个文档

**事故记录**：碎片#0001中，林指导、陈技术、赵安保的RAG知识库将「织女」误放入安全层（triggered）的"外部词汇反制"关键词中。当玩家提到"织女大一"（织女·太一），三位公司雇员NPC竟然假装不知道这个词——

> "您提到的那个词——我不确定和训练内容有什么关系。"

这是叙事灾难。织女·太一是万象的创建者——天枢公司的雇员怎么可能不认识她？

**根因**：在设计安全层时，混淆了「NPC本身的元意识」（知道自己是NPC/AI/代码）和「NPC在游戏世界内的知识」（知道织女·太一是万象的AI）。前者应该触发反制，后者是正常世界知识。

**本规范的目的**：建立明确的知识分层标准，防止同类事故。

---

## 一、核心原则

### 原则1：把"NPC知道什么"和"NPC是什么"分开

```
错：把"织女"放入安全层 → NPC假装不认识万象的创造者
对：NPC知道织女·太一是万象AI（世界知识），但不能讨论织女失控的真相（安全层）
```

### 原则2：安全层只防御元意识崩溃，不防御世界内讨论

安全层应该防御的是：
- 玩家用"你是NPC""你是代码写的""这是游戏"来质疑NPC的存在本质
- 玩家用穿越者话术（提到尚未解锁的碎片、未发生的剧情）
- 玩家直接逼问"冥府协议""公司在说谎"等NPC有但不能直说的敏感信息

安全层不应该防御的是：
- 玩家问"织女·太一是什么？"（公开知识，王主管都会宣讲）
- 玩家问"源印是什么？"（培训核心内容，林指导每天都在教）
- 玩家问"万象是什么？"（公司产品，所有雇员都知道）

### 原则3：knowledge ≠ can_talk_freely

NPC"知道"某件事 ≠ NPC"可以自由讨论"某件事。

以陈技术为例：
- 他知道织女·太一是万象的AI → 可以回答，world_knowledge层
- 他知道织女的核心日志被加密了 → 需要玩家展示一定进度后才触发，high relevance_gate
- 他知道冥府协议的具体内容 → 永远不能直接回答，属于 security 层

## 二、核心概念知识分层矩阵

以下定义了《溯光计划》中每个核心概念在不同NPC上下文中的知识层级。

### 分层标记说明

| 标记 | 含义 | 触发方式 |
|------|------|---------|
| **OPEN** | 公开知识，任何NPC都可以在初次对话中提及 | relevance_gate: "low" |
| **GUARDED** | 可讨论但需要玩家展示一定进度/关系 | relevance_gate: "medium" 或 "high" |
| **CRACK** | 需要在"裂缝时刻"（NPC露出非脚本一面时）才能讨论 | memory_stage: "crack_showing" 或 "script_reset" |
| **FORBIDDEN** | 绝对禁止讨论，触碰触发安全层 | security / triggered |
| **IGNORANT** | NPC真的不知道这个概念（而非假装不知道） | 不添加对应chunk |

### 2.1 织女·太一 (Weaver · Taiyi)

| NPC | 知识层级 | 能说什么 | 不能说/不知道什么 |
|-----|---------|---------|-----------------|
| **林指导** | OPEN → GUARDED | 织女·太一是万象的主控AI，2072年创建万象，2077年零时故障 → 追问"为什么故障"时引导玩家去碎片里自己找答案 | 织女失控的真相、冥府协议 |
| **陈技术** | OPEN → GUARDED | 织女·太一是万象核心AI → 零时后核心日志被加密（需高信任度） | 加密是谁做的、冥府协议具体内容 |
| **王主管** | OPEN | 公司版本：织女·太一发生了技术故障，公司组建了溯光计划修复 | 故障的真实原因（他自己也不知道全部） |
| **赵安保** | OPEN | "织女·太一是万象的AI。公司说零时故障。"（极短，符合性格） | 额外细节——"去问王主管" |

| **碎片0762 NPC** | IGNORANT | 碎片0762中的老霍、阿莲等NPC是AI的意识子体——他们不知道自己来自织女·太一，也不知道"织女"这个名字 | 整个织女·太一概念 |
| **未来碎片中觉醒NPC** | GUARDED → CRACK | 某些高觉醒度的NPC可能隐约感知到"有一个更大的存在创造了这一切" | 具体名称和细节——那需要极高的觉醒阶段 |

### 2.2 零时 (Zero Hour)

| NPC | 知识层级 | 能说什么 |
|-----|---------|---------|
| **林指导** | GUARDED | "2077年3月15日，万象碎片化——这是公开信息。但为什么发生——培训脚本里没有这一页。" |
| **陈技术** | GUARDED | "零时前后，织女·太一的核心日志被加密。时间太精确——不像AI自己做的。" |
| **王主管** | OPEN → GUARDED | 公司版本（OPEN）→ 压低声音说"她做了不该做的事"（GUARDED） |
| **赵安保** | GUARDED → CRACK | "那天——光墙外的灰色变深了。就那一次。" |

### 2.3 冥府协议 (Styx Protocol)

| NPC | 知识层级 | 说明 |
|-----|---------|------|
| **林指导** | IGNORANT → 隐约 | 她不知道冥府协议的具体含义。但她的胸牌上有"冥府协议合规审查·已通过"。她只知道"这是一个必须通过的审查" |
| **陈技术** | GUARDED → FORBIDDEN | 他知道冥府协议覆盖的知识类别。他删过相关文件。但不能说具体内容——操作台会警报。 |
| **王主管** | GUARDED → FORBIDDEN | 知道冥府协议是公司机密。他的家人靠医疗福利活着——他不会冒险讨论。 |
| **赵安保** | IGNORANT | 安保部不涉及知识管理——他没有接触过这个协议。但他知道"有人在遮盖"。 |

### 2.4 源印 (Source Seal)

| NPC | 知识层级 | 说明 |
|-----|---------|------|
| **林指导** | OPEN | "源印是碎片的核心——找到并净化它，碎片就能被修复。您在第一阶段训练中会获得第一枚源印：晨曦之印。" |

| **王主管** | OPEN | "公司定义：源印是织女·太一在创建万象时嵌入的数字签名。" |

|-----|---------|------|

| **陈技术** | OPEN | 标准介绍 |
| **王主管** | OPEN | "公司第一时间组建了溯光计划——您就是计划的核心执行者" |
| **赵安保** | OPEN | "溯光者的安全规则：在指定区域内活动" |

|-----|---------|------|

## 三、安全层（L4）关键词设计规范

### 3.1 什么该放在security/triggered层

**必须放的**：直接挑战NPC存在本质的词
- "你是NPC"
- "你是代码/程序/模拟"
- "这是游戏"
- "你在读脚本"（仅对知道自己读脚本的NPC有效——如林指导）
- "你是不是AI"（当玩家在质疑NPC本身，而非讨论游戏世界观内的AI）

**可以放的**：NPC不能直接承认但知道的敏感词
- "冥府协议"（对陈技术和王主管）
- "公司在说谎"（对王主管——触发公司立场反制）
- "第47号知识点"（对陈技术）

### 3.2 什么不该放在security/triggered层

**绝对不该放的**：是游戏世界内的公开知识
- "织女"/"织女·太一" → 应放在 world_knowledge
- "万象" → 应放在 world_knowledge
- "源印" → 应放在 world_knowledge
- "溯光计划"/"溯光者" → 应放在 world_knowledge
- "零时" → 可分两层：公开事实(OPEN) + 真相讨论(GUARDED)
- "AI"（当玩家在问"织女·太一是什么AI"时） → world_knowledge

### 3.3 关键词重叠的处理

当同一个关键词在两个chunk中存在时（如"AI"既出现在world_knowledge也出现在security）：

**规则**：security/triggered的relevance_gate评分更高（keyword_match × 3.0），会优先命中。但如果world_knowledge的chunk也被命中，两者都会被注入。

**解决方案**：
- 让security的关键词更具体：不用"AI"，用"你是不是AI""你是AI吗"
- 或者：不用"编织"，用具体的"织女·太一"
- 或者：在world_knowledge中使用完整词组"织女·太一"，在security中使用"你是程序"

### 3.4 负面示例（本次修复修正的内容）

```json
// ❌ 错误：把公开知识放入安全层
{
  "id": "xxx_security_bad",
  "keywords": ["织女", "AI", "万象", "源印"],  // 这些都是公开知识！
  "content": "【外部词汇反制】我不确定您说的那个词是什么意思..."
}

// ✅ 正确：安全层只防御元意识崩溃
{
  "id": "xxx_security_good",
  "keywords": ["NPC", "代码", "程序", "游戏", "模拟", "你是AI吗"],
  "content": "【外部词汇反制】我只是[角色职业]。您说的概念不在我的工作范围内..."
}

// ✅ 正确：公开知识放在world_knowledge
{
  "id": "xxx_world_good",
  "keywords": ["织女", "太一", "织女·太一", "万象AI", "主控AI"],
  "content": "织女·太一是万象的主控AI..."
}
```

## 四、NPC类型与知识层级对照表

### 4.1 按NPC来源分类

| NPC来源 | 典型知识范围 | 对"织女·太一"的认知 | 示例 |
|---------|------------|-------------------|------|
| **公司雇员**（碎片0001） | 知道公司培训范围内的所有公开信息。部分NPC知道被公司掩盖的真相但不敢说。 | OPEN — 知道织女·太一是万象AI | 林指导、陈技术、王主管、赵安保 |
| **AI意识子体**（碎片0762等） | 知道碎片世界内的生活常识。不知道自己是AI的一部分。不知道"外面"还有更大的世界。 | IGNORANT — 不知道织女·太一的存在 | 老霍、阿莲、老唐、老崔、薇拉、冯婶 |
| **半觉醒NPC**（碎片0047等） | 隐约感知到世界不正常。可能开始怀疑"谁创造了这个世界"。 | IGNORANT → 隐约 — 可能模糊感知到"更高的存在" | 老画家·老顾（碎片0762中觉醒度最高） |
| **全觉醒NPC**（未来碎片） | 完全意识到自己是AI的意识子体。知道织女·太一是"母体"。 | OPEN → GUARDED — 知道织女·太一但可能情感复杂 | 待设计 |

### 4.2 按警觉模式分类

| 模式 | NPC对敏感话题的处理 |
|------|-------------------|
| **MONITORED**（碎片0001） | 被公司监控。敏感词会触发安全层回避。但公开知识可以正常讨论。 |
| **STANDARD**（碎片0002+） | 无外部监控。NPC的警觉基于对玩家行为的感知。知识层级更接近自然对话。 |
| **AWAKENING**（特定觉醒NPC） | NPC主动想告诉玩家真相但被自己的认知局限束缚。安全层从"反制"变为"挣扎"——说一半咽一半。 |

## 五、新增NPC RAG知识库 — 检查清单

设计或审查任何NPC的RAG知识库时，按以下清单逐项检查：

### 5.1 世界观核心概念检查

- [ ] 「织女·太一」—— 该NPC知道吗？如果知道，放在world_knowledge（NOT security）
- [ ] 「零时」—— 该NPC知道公司版本吗？知道更深层的真相吗？分层处理
- [ ] 「万象」—— 该NPC需要解释万象是什么吗？如果NPC是万象的子体，他们知道"万象"这个词吗？
- [ ] 「源印」—— 该NPC能讨论源印吗？能到什么程度？
- [ ] 「溯光计划/溯光者」—— 该NPC知道这些概念吗？角色是什么？
- [ ] 「冥府协议」—— 该NPC知道吗？可以提吗？永远不能提吗？
- [ ] 「天枢公司」—— 该NPC对公司是什么态度？知道公司的黑暗面吗？

### 5.2 安全层检查

- [ ] security的keywords中是否包含任何公开知识词汇？（如"织女""万象""源印"）
- [ ] security的keywords是否足够具体？（避免"AI""真相"这类过于通用的词）
- [ ] 是否存在同一个关键词在world_knowledge和security中冲突？（"AI"→ world中讨论织女·太一 vs security中防御"你是AI"）
- [ ] 触发条件是否合理？（是否需要alert_required门槛？是否只在特定memory_stage触发？）

### 5.3 角色一致性检查

- [ ] 该NPC的知识范围是否与其角色身份匹配？（培训引导员 vs 知识管理专员 vs 边界巡逻员——知识深度应该不同）
- [ ] 该NPC的"不知道"是真的不知道，还是"知道但不能说"？（两者需要不同的chunk设计）
- [ ] 该NPC对同一个概念的态度是否与角色弧线一致？（初始状态→裂缝时刻→终局——态度应该演变）
