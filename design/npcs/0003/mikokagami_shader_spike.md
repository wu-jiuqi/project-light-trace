## 一、验证目标

确认在 Godot 4.x 引擎中，2D 俯视角纸艺风格下实现**实时反射 + 0.3秒延迟帧缓冲 + 纸纹理叠加 + 凸面镜变形**的 Shader 方案的技术可行性。

---

## 二、验收标准

| # | 条件 | 判定方式 |
|---|------|---------|
| 1 | 巫女镜面能够实时捕获玩家Sprite所在区域的像素 | 玩家在镜前移动→镜面同步变化 |
| 2 | 反射延迟约0.3秒（环形帧缓冲3帧@10fps） | 计时器测量帧差 |
| 3 | 叠加的纸纹理（粗糙度贴图+淡蓝光晕）不破坏反射可读性 | 肉眼审查——玩家能辨认镜中是自己 |
| 4 | 凸面镜变形（左右拉伸5%）效果可见但不过度 | 反射中角色比原Sprite略宽但不失真 |
| 5 | 性能达标：FPS不低于目标帧率90% | 开/关Shader场景FPS对比 |

---

## 三、方案矩阵

### 方案A：SubViewport + Camera2D 实时捕获（理想方案）

```
技术路径：
Godot SubViewport → Camera2D跟随玩家 → 
RenderTexture捕获 → 环形帧缓冲(
  buffer[0..2], write_idx循环, 每333ms写入一帧) → 
纸纹Shader(粗糙度贴图+淡蓝冷光晕 #A0C8E8混合) → 
凸面镜UV偏移(UV.x += (UV.x - 0.5) * 0.05) → 
输出到巫女镜面TextureRect
```

| 项 | 值 |
|----|-----|
| 预估工时 | 4-6 小时 |
| 优点 | 完全符合设计意图：实时 + 延迟 + 纸纹理 + 变形 |
| 风险 | Godot 2D中SubViewport性能未知；10fps缓冲可能导致"卡顿感" |
| 验证项 | SubViewport在2D场景的可用性、RenderTexture → Shader的管线 |

### 方案B：定时截图 + Shader后处理（降级方案一）

```
技术路径：
每隔500ms调用 get_viewport().get_texture().get_image() 快照 → 
resize到镜面分辨率 → 直接设为镜面TextureRect的texture → 
Shader层叠加纸纹理和微变形
```

| 项 | 值 |
|----|-----|
| 预估工时 | 3-4 小时 |
| 优点 | 实现简单，性能可控，延迟自然产生 |
| 缺点 | 0.5秒延迟略长于设计要求的0.3秒；快照频率低可能导致"跳帧" |
| 验证项 | get_image()在2D中的性能开销、快照分辨率 vs 视觉效果 |

### 方案C：预渲染关键帧序列（降级方案二）

```
技术路径：
根据玩家外观（进入碎片时预设），加载6-8帧预渲染镜面反射帧 → 
循环播放帧序列 + Shader纸纹理叠加 + 变形
```

| 项 | 值 |
|----|-----|
| 预估工时 | 2-3 小时（不含美术资产） |
| 优点 | 无运行时反射开销，帧序列质量可控 |
| 缺点 | 放弃实时反射——镜面不会"实时映出玩家的动作" |
| 验证项 | 预渲染帧序列的接受度测试（美术侧） |

### 方案D：静态波纹Shader（最低降级方案）

```
技术路径：
复用枯井反射面Shader → 
玩家靠近时增加波纹扰动 → 
模糊的玩家色块轮廓 + 纸纹理 + 淡蓝光晕
```

| 项 | 值 |
|----|-----|
| 预估工时 | 1-2 小时 |
| 优点 | 现有技术基础，几乎零风险 |
| 缺点 | 不能"照出玩家"——损失核心设计意图 |
| 验证项 | 接受度测试——此方案极大弱化"镜中倒影"叙事 |

## 四、Spike 执行计划

```
Day 1（2h）：搭建测试场景
  - 创建独立Godot场景（非碎片0003主场景）
  - 放置巫女Sprite + 玩家Sprite
  - 实现方案A基础管线（不含延迟缓冲和纸纹理）

Day 2（2h）：方案A验证
  - 加入环形帧缓冲 → 测量延迟精度
  - 加入纸纹理Shader → 效果审查
  - 加入凸面镜UV偏移 → 效果审查
  - 性能测试（开/关Shader FPS对比）

Day 3（1-2h）：方案A验收/降级决策
  - 方案A验收：达到验收标准→结束Spike
  - 方案A不达标：降级到方案B→重新验证
  - 结论文档化→反馈设计团队
```

## 五、判定规则

| 场景 | 行动 |
|------|------|
| 方案A 达到5条验收标准 | ✅ 采纳方案A，进入正式实现 |
| 方案A 验收标准#2不达标但#1达标 | 降级为方案B——接受0.5秒延迟 |
| 方案A 验收标准#1不达标 | 降级为方案C——预渲染序列 + Shader |
| 方案C 美术效果不可接受 | 🔴 降级为方案D——保底方案。需重新评估碎片的视觉叙事是否仍满足"镜中倒影"核心体验 |
